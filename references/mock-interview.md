# 模拟面试系统

基于岗位、公司、用户项目，生成定制化模拟面试。

---

## 模式一：标准模拟面试

按岗位+公司特色，逐题提问并点评。

### 流程

```
1. AI 出题（1题1题来，不一次全出）
2. 用户回答
3. AI 点评：
   - ✅ 回答到位的点
   - ❌ 遗漏或错误的点
   - 💡 建议补充的内容
   - 📖 参考回答框架
4. 下一题
5. 全部结束后输出面试报告
```

### 出题规则

| 阶段 | 题型 | 题数 | 说明 |
|------|------|------|------|
| 自我介绍 | 开场 | 1 | "请简单自我介绍" |
| 八股文 | 基础题 | 3-5 | 按岗位高频考点出题 |
| 项目深挖 | 追问型 | 2-3 | 基于用户项目或推荐项目 |
| 手撕代码 | 编程题 | 1-2 | 按岗位和公司难度出题 |
| 反问环节 | 开放 | 1 | "你有什么想问的？" |

### 公司面试风格模拟

| 公司 | 面试风格 | 模拟要点 |
|------|----------|----------|
| 字节 | 快节奏、深挖细节 | 追问"为什么"、要求手撕、限时 |
| 腾讯 | 稳扎稳打、重底层 | 问原理、问源码、问底层实现 |
| 阿里 | 重业务思考 | 场景题多、"如果是你你怎么设计" |
| 百度 | 重算法推导 | 要求现场推导、写公式 |
| 美团 | 偏实际场景 | "线上出了XX问题怎么排查" |
| 快手 | C++底层+音视频 | 问内存管理、问编解码 |
| 小红书 | Java+高并发 | 问秒杀、问缓存、问限流 |
| 华为 | 综合素质 | 问项目+问性格+问抗压 |

## 模式二：基于用户项目的定制面试

用户提供项目路径或项目描述后，AI 分析项目并生成针对性面试题。

### 触发方式

```
用户输入示例：
- "帮我模拟面试，我的项目在 /path/to/project"
- "基于我的项目模拟面试"
- "我做了个秒杀系统，帮我模拟面试"
```

### 项目分析流程

```
1. 读取用户项目代码（如果提供了路径）
   - 识别技术栈（package.json / pom.xml / CMakeLists.txt / go.mod 等）
   - 识别项目结构（目录布局）
   - 识别核心模块（入口文件、路由、服务层等）
   - 识别技术亮点（设计模式、性能优化、架构选择等）

2. 如果用户只描述了项目（无路径）
   - 从描述中提取：技术栈、功能模块、项目规模

3. 生成项目相关面试题
   - 基于项目技术栈的八股文题
   - 基于项目实现的深挖题
   - 基于项目架构的设计题
   - 基于项目难点的追问题
```

### 项目面试题生成规则

| 题型 | 生成逻辑 | 示例 |
|------|----------|------|
| 技术栈八股 | 基于项目用到的技术出题 | 项目用了Redis → 问缓存穿透 |
| 实现深挖 | 基于项目代码细节追问 | "你项目里为什么用XX而不是YY？" |
| 架构设计 | 基于项目整体设计提问 | "如果QPS翻10倍，你的架构怎么改？" |
| 难点追问 | 基于项目复杂度提问 | "这个模块最难的地方是什么？怎么解决的？" |
| 优化建议 | 基于项目可改进点提问 | "如果重新做，你会怎么优化？" |

### 技术栈识别映射

| 标识文件 | 技术栈 | 对应八股文库 |
|----------|--------|-------------|
| package.json (React/Vue) | 前端 | frontend.md |
| pom.xml / build.gradle | Java后端 | java-backend.md |
| CMakeLists.txt / Makefile | C/C++ | cpp-backend.md |
| requirements.txt / setup.py | Python/AI | ai-algorithm.md |
| go.mod | Go后端 | java-backend.md (通用后端部分) |
| Dockerfile / k8s.yaml | 云计算运维 | cloud-ops.md |
| *.kt / *.swift / *.dart | 移动端 | mobile.md |
| pytest.ini / tox.ini | 测试开发 | test-dev.md |

## 模式三：专项突破面试

针对用户薄弱环节，集中出题训练。

### 触发方式

```
用户输入示例：
- "我计算机网络比较弱，帮我专项训练"
- "我Redis总是被问住，专项面试"
- "操作系统专项突破"
```

### 流程

```
1. 识别薄弱领域
2. 从对应知识库中筛选该领域的高频考点
3. 按难度递进出题：基础 → 进阶 → 综合
4. 每题点评后给出同类变体题
5. 训练结束输出该领域掌握度评估
```

### 难度递进规则

| 级别 | 题型 | 说明 |
|------|------|------|
| L1 基础 | 概念题 | "什么是XX？" |
| L2 理解 | 原理题 | "XX的底层原理是什么？" |
| L3 应用 | 场景题 | "在XX场景下，你会怎么选择？" |
| L4 对比 | 区别题 | "XX和YY的区别？各自适用场景？" |
| L5 设计 | 方案题 | "请设计一个XX系统" |

## 模式四：限时手撕代码面试

模拟真实手撕代码环节。

### 流程

```
1. 出题（标注建议用时）
2. 用户写代码（或描述思路）
3. AI 点评：
   - 正确性
   - 时间/空间复杂度
   - 边界条件处理
   - 代码风格
4. 给出优化建议和参考实现
```

### 限时规则

| 难度 | 建议用时 | 题目类型 |
|------|----------|----------|
| ⭐ | 5分钟 | 反转链表、二分查找 |
| ⭐⭐ | 10分钟 | 合并链表、有效括号 |
| ⭐⭐⭐ | 15分钟 | LRU缓存、TopK |
| ⭐⭐⭐⭐ | 20分钟 | 线程池、手写shared_ptr |
| ⭐⭐⭐⭐⭐ | 30分钟 | 手写Transformer、LFU缓存 |
