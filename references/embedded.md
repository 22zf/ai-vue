# 嵌入式面试八股文库

## 一、C语言基础

### 指针与数组 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：数组名是首元素地址的常量（sizeof除外），指针是存储地址的变量，两者在传参时退化但本质不同。
**常见面试问法**：
- "数组和指针有什么区别？"
- "指针运算的规则是什么？"
**回答框架**：
1. 数组名：是地址常量不可赋值，sizeof(数组)返回整个数组大小，&数组名返回数组指针
2. 指针：是变量可赋值，sizeof(指针)返回指针大小（4/8字节），可做加减运算
3. 指针运算：p+n实际偏移n*sizeof(*p)字节，指针相减得到元素个数差
4. 传参退化：数组作为函数参数时退化为指针，丢失长度信息，需额外传长度参数
**延伸考点**：函数指针 → 结构体对齐

### 内存管理 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：栈由编译器自动管理（函数局部变量），堆由程序员手动管理（malloc/free），嵌入式开发中堆的使用需极其谨慎。
**常见面试问法**：
- "栈和堆有什么区别？"
- "malloc分配的内存不free会怎样？"
**回答框架**：
1. 栈：自动分配释放、速度快、空间有限（通常1-8MB）、生长方向从高到低
2. 堆：手动分配释放、速度慢（需查找空闲链表）、空间大、容易产生碎片
3. 内存泄漏：malloc后未free导致内存持续占用，嵌入式系统中可能导致系统崩溃
4. 嵌入式实践：尽量避免动态分配，使用静态分配+内存池，确保确定性
**延伸考点**：volatile → RTOS内存管理

### 位操作 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：位操作是嵌入式开发的核心技能，用于寄存器配置、标志位管理和通信协议解析，必须熟练掌握位掩码技巧。
**常见面试问法**：
- "怎么把一个寄存器的第3位置1，第5位清0？"
- "位域有什么注意事项？"
**回答框架**：
1. 置位：reg |= (1 << n)，将第n位置1
2. 清位：reg &= ~(1 << n)，将第n位清0
3. 取反：reg ^= (1 << n)，将第n位取反
4. 判断：if (reg & (1 << n))，判断第n位是否为1
5. 位域：struct { unsigned int bit0:1; unsigned int bit1:3; }，注意位域对齐和可移植性问题
**延伸考点**：结构体对齐 → 寄存器操作

### 结构体对齐与sizeof [频率: ⭐⭐⭐⭐⭐]
**核心要点**：结构体按成员最大对齐倍数对齐，编译器在成员间插入填充字节，sizeof计算需考虑对齐规则，影响内存布局和通信协议。
**常见面试问法**：
- "结构体怎么对齐的？怎么计算sizeof？"
- "怎么让结构体按1字节对齐？"
**回答框架**：
1. 对齐规则：每个成员偏移量必须是其自身对齐倍数的整数倍，结构体总大小是最大对齐倍数的整数倍
2. 计算示例：struct { char a; int b; short c; } → 1+3(填充)+4+2+2(填充) = 12字节
3. 调整顺序：struct { int b; short c; char a; } → 4+2+1+1(填充) = 8字节，节省空间
4. 修改对齐：#pragma pack(1)或__attribute__((packed))按1字节对齐，用于通信协议结构体
**延伸考点**：位操作 → 大端小端

### volatile/const/static [频率: ⭐⭐⭐⭐⭐]
**核心要点**：volatile告诉编译器不要优化对变量的访问（每次从内存读取），const表示不可修改，static控制作用域和生命周期。
**常见面试问法**：
- "volatile有什么用？哪些场景必须用？"
- "static在C语言中有哪些用法？"
**回答框架**：
1. volatile：禁止编译器缓存变量值，每次从内存读取。用于中断共享变量、多线程共享变量、硬件寄存器映射
2. const：定义常量、修饰指针（const在*左边修饰值，在*右边修饰指针）、函数参数保护
3. static：修饰局部变量（生命周期延长到程序结束）、修饰全局变量/函数（限制作用域为当前文件）
4. 组合使用：volatile const uint32_t *reg = (uint32_t *)0x40021000; 寄存器只读但不优化
**延伸考点**：内存管理 → 中断处理

### 函数指针与回调函数 [频率: ⭐⭐⭐⭐]
**核心要点**：函数指针存储函数入口地址，回调函数通过函数指针实现"调用者定义行为"的设计模式，是驱动框架和事件系统的基石。
**常见面试问法**：
- "函数指针怎么用？什么是回调函数？"
- "回调函数在嵌入式开发中有什么应用？"
**回答框架**：
1. 函数指针：int (*func_ptr)(int, int) = &add; 通过func_ptr(1,2)调用
2. 回调函数：将函数指针作为参数传给另一个函数，由被调方在适当时机调用
3. 嵌入式应用：中断回调、驱动注册回调、事件通知机制、状态机实现
4. typedef简化：typedef void (*callback_t)(int event); 提高可读性
**延伸考点**：指针与数组 → RTOS任务间通信

### 大端小端与字节序转换 [频率: ⭐⭐⭐⭐]
**核心要点**：大端模式高位字节存低地址（网络序），小端模式高位字节存高地址（x86/ARM默认），通信协议必须统一字节序。
**常见面试问法**：
- "什么是大端小端？怎么判断当前系统是大端还是小端？"
- "网络字节序和主机字节序有什么区别？"
**回答框架**：
1. 大端（Big-Endian）：高位字节存低地址，符合人类阅读习惯，网络协议使用大端
2. 小端（Little-Endian）：高位字节存高地址，x86/ARM默认，运算效率高
3. 判断方法：int a = 1; if (*(char*)&a == 1) 则为小端
4. 转换函数：htons/htonl（主机转网络）、ntohs/ntohl（网络转主机）
**延伸考点**：结构体对齐 → 通信协议

## 二、单片机/处理器

### ARM架构 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：ARM Cortex-M用于微控制器（实时性），Cortex-A用于应用处理器（跑Linux），Cortex-R用于实时处理器，三者定位不同。
**常见面试问法**：
- "Cortex-M和Cortex-A有什么区别？"
- "ARM的寄存器有哪些？"
**回答框架**：
1. Cortex-M：M0/M3/M4/M7，无MMU、NVIC中断控制器、低功耗、跑裸机/RTOS
2. Cortex-A：A7/A53/A72，有MMU、支持Linux/Android、高性能、多核
3. Cortex-R：硬实时、安全关键场景（汽车/工业）
4. 核心寄存器：R0-R12通用寄存器、SP栈指针、LR链接寄存器、PC程序计数器、CPSR状态寄存器
**延伸考点**：中断处理 → 启动流程

### ARM异常与中断处理 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：中断是异常的一种，ARM通过向量表定位中断服务函数，中断处理需保存现场→处理→恢复现场，优先级由NVIC管理。
**常见面试问法**：
- "ARM中断处理的流程是什么？"
- "中断中能不能调用printf？"
**回答框架**：
1. 中断流程：中断触发→硬件保存现场（压栈CPSR/PC/LR）→查向量表跳转ISR→执行中断服务→恢复现场（出栈）→返回
2. NVIC：嵌套向量中断控制器，支持中断优先级、中断嵌套、中断屏蔽
3. ISR注意事项：不能调用阻塞函数（printf/malloc）、不能使用浮点运算（M0/M3）、快速执行
4. 中断与任务通信：ISR中发送信号量/事件标志，任务中处理具体逻辑
**延伸考点**：RTOS任务调度 → 优先级翻转

### 启动流程（Bootloader） [频率: ⭐⭐⭐⭐⭐]
**核心要点**：MCU上电后从向量表获取复位向量→执行Reset_Handler→初始化系统时钟/堆栈/BSS段→跳转main()，理解启动流程是调试硬件问题的基础。
**常见面试问法**：
- "STM32上电后的启动流程是什么？"
- "Bootloader的作用是什么？"
**回答框架**：
1. 上电复位：CPU从0x00000000（或向量表起始地址）取初始SP和复位向量PC
2. Reset_Handler：调用SystemInit()配置时钟→初始化.data段（从Flash拷贝到RAM）→清零.bss段→调用__main→跳转main()
3. Bootloader：在操作系统运行前初始化硬件、加载内核、传递启动参数
4. U-Boot流程：start.S汇编初始化→board_init_f→重定位→board_init_r→加载内核
**延伸考点**：ARM架构 → Linux内核裁剪

### STM32架构与时钟树 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：STM32时钟树通过分频/倍频为各外设提供时钟，合理配置时钟树是低功耗和外设正常工作的前提。
**常见面试问法**：
- "STM32的时钟树怎么配置？"
- "HSI和HSE有什么区别？"
**回答框架**：
1. 时钟源：HSI（内部RC，8/16MHz，精度低）、HSE（外部晶振，4-26MHz，精度高）、LSI/LSE（低速，RTC/看门狗）
2. 时钟树：时钟源→PLL倍频→SYSCLK→AHB分频→APB1/APB2分频→各外设时钟
3. 常见配置：HSE(8MHz)→PLL×9→SYSCLK(72MHz)→AHB(72MHz)→APB1(36MHz)/APB2(72MHz)
4. 低功耗：关闭未使用外设时钟、降低主频、使用STOP/STANDBY模式
**延伸考点**：GPIO配置 → 低功耗设计

### GPIO与中断配置 [频率: ⭐⭐⭐⭐]
**核心要点**：GPIO可配置为输入/输出/复用/模拟模式，外部中断通过EXTI线触发，需配置触发方式和优先级。
**常见面试问法**：
- "GPIO有哪些模式？"
- "外部中断怎么配置？"
**回答框架**：
1. GPIO模式：输入（浮空/上拉/下拉）、输出（推挽/开漏）、复用（AF推挽/AF开漏）、模拟
2. 推挽vs开漏：推挽可主动输出高低电平，开漏只能输出低电平需外接上拉
3. EXTI配置：配置GPIO为输入→映射到EXTI线→配置触发方式（上升沿/下降沿/双边沿）→配置NVIC优先级→编写ISR
4. 消抖：硬件消抖（RC滤波）+软件消抖（延时采样/定时器消抖）
**延伸考点**：STM32时钟树 → 定时器

### 定时器（PWM/输入捕获） [频率: ⭐⭐⭐⭐]
**核心要点**：STM32定时器分为基本/通用/高级定时器，PWM输出和输入捕获是最常用的两种功能，分别用于驱动和测量。
**常见面试问法**：
- "PWM的频率和占空比怎么计算？"
- "输入捕获怎么测量脉宽？"
**回答框架**：
1. 定时器分类：基本定时器（TIM6/7仅定时）、通用定时器（TIM2-5 PWM/捕获）、高级定时器（TIM1/8 电机控制）
2. PWM输出：频率 = TIM_CLK / ((PSC+1)×(ARR+1))，占空比 = CCR / (ARR+1)
3. 输入捕获：捕获上升沿/下降沿时刻的计数器值，两次捕获差值×时钟周期=脉宽
4. 应用：PWM驱动LED/电机/舵机，输入捕获测频率/占空比/编码器
**延伸考点**：GPIO配置 → ADC/DAC

## 三、通信协议

### UART/USART [频率: ⭐⭐⭐⭐⭐]
**核心要点**：UART是异步串行通信协议，全双工、点对点，只需TX/RX两根线，是嵌入式最基础的通信方式。
**常见面试问法**：
- "UART的通信格式是什么？"
- "波特率怎么配置？常见的波特率有哪些？"
**回答框架**：
1. 通信格式：起始位(1)→数据位(5-9)→校验位(0/1)→停止位(1/1.5/2)，空闲为高电平
2. 波特率：每秒传输的位数，常用9600/115200，双方必须一致
3. USART vs UART：USART多了一个同步时钟线SCLK，支持同步通信和智能卡模式
4. 常见问题：波特率不匹配导致乱码、中断接收丢数据（需DMA或环形缓冲区）
**延伸考点**：SPI → I2C

### SPI [频率: ⭐⭐⭐⭐⭐]
**核心要点**：SPI是同步串行通信协议，全双工、高速，四线制（SCLK/MOSI/MISO/CS），支持多从设备但每从设备需独立CS线。
**常见面试问法**：
- "SPI的四根线分别是什么作用？"
- "SPI的四种模式有什么区别？"
**回答框架**：
1. 四线：SCLK时钟、MOSI主出从入、MISO主入从出、CS片选
2. 四种模式：由CPOL（时钟极性）和CPHA（时钟相位）决定，CPOL=0/1空闲低/高，CPHA=0/1第一/二个边沿采样
3. Mode0（CPOL=0,CPHA=0）最常用，空闲低电平，上升沿采样
4. 多从设备：每个从设备独立CS线，或菊花链连接（数据依次移位）
**延伸考点**：I2C → UART

### I2C [频率: ⭐⭐⭐⭐⭐]
**核心要点**：I2C是同步半双工通信协议，两线制（SCL/SDA），通过7/10位地址寻址，支持多主多从，有仲裁和应答机制。
**常见面试问法**：
- "I2C怎么寻址的？"
- "I2C的仲裁机制是什么？"
**回答框架**：
1. 通信流程：主机发送起始位→发送从机地址+读写位→从机应答→数据传输→应答→停止位
2. 地址：7位地址（0x00-0x7F）或10位地址，同一总线上不能有地址冲突
3. 仲裁：多主机同时发送时，SDA线"线与"特性，发送0的主机获胜，发送1检测到0的主机退出
4. 与SPI对比：I2C线少但速度慢（标准100K/快速400K），SPI线多但速度快（可达几十MHz）
**延伸考点**：SPI → CAN

### CAN [频率: ⭐⭐⭐⭐]
**核心要点**：CAN是差分信号的串行总线协议，支持多主通信，通过仲裁机制解决冲突，具有强大的错误检测能力，广泛用于汽车和工业。
**常见面试问法**：
- "CAN的仲裁机制是什么？"
- "CAN的标准帧和扩展帧有什么区别？"
**回答框架**：
1. 物理层：差分信号（CAN_H/CAN_L），显性(0)覆盖隐性(1)，抗干扰能力强
2. 仲裁：标识符逐位仲裁，ID越小优先级越高（显性位优先），非破坏性仲裁
3. 帧类型：数据帧（传输数据）、远程帧（请求数据）、错误帧、过载帧
4. 标准帧vs扩展帧：标准帧11位ID，扩展帧29位ID，通过IDE位区分
**延伸考点**：I2C → Modbus

### Modbus [频率: ⭐⭐⭐]
**核心要点**：Modbus是应用层通信协议，基于主从模式，RTU模式用二进制传输效率高，TCP模式基于以太网，是工业自动化的事实标准。
**常见面试问法**：
- "Modbus RTU和Modbus TCP有什么区别？"
- "Modbus的功能码有哪些？"
**回答框架**：
1. 传输模式：ASCII（可读但慢）、RTU（二进制高效）、TCP（基于以太网）
2. 主从模式：主机发起请求，从机响应，从机不能主动发起通信
3. 功能码：01读线圈、02读离散输入、03读保持寄存器、04读输入寄存器、05/06写单个、15/16写多个
4. CRC校验：RTU模式使用CRC16校验，保证数据完整性
**延伸考点**：CAN → MQTT

### MQTT [频率: ⭐⭐⭐⭐]
**核心要点**：MQTT是轻量级发布/订阅模式的消息协议，基于TCP，适合物联网场景，支持QoS等级和遗嘱消息。
**常见面试问法**：
- "MQTT的发布/订阅模式是什么？"
- "MQTT的QoS等级有哪些？"
**回答框架**：
1. 架构：发布者→Broker（代理）→订阅者，发布者和订阅者解耦
2. QoS等级：QoS0（至多一次，可能丢失）、QoS1（至少一次，可能重复）、QoS2（恰好一次，不丢不重）
3. Topic：分层结构（sensor/temperature/room1），支持通配符（+单层/#多层）
4. 遗嘱消息：客户端异常断开时Broker自动发布其遗嘱消息，通知其他客户端
**延伸考点**：Modbus → 物联网终端项目

## 四、RTOS

### 任务调度原理 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：RTOS采用基于优先级的抢占式调度，高优先级任务就绪时立即抢占低优先级任务，同优先级采用时间片轮转。
**常见面试问法**：
- "RTOS的任务调度算法是什么？"
- "抢占式调度和协作式调度有什么区别？"
**回答框架**：
1. 抢占式调度：高优先级就绪立即抢占当前任务，保证实时性
2. 协作式调度：任务主动让出CPU才切换，实时性差但简单
3. 时间片轮转：同优先级任务轮流执行，每个任务执行一个时间片
4. 调度时机：任务创建/删除、中断退出、任务挂起/恢复、延时到期
**延伸考点**：任务状态 → 优先级翻转

### 任务状态 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：RTOS任务有就绪/运行/阻塞/挂起四种状态，状态转换由调度器和任务操作触发，理解状态机是调试任务问题的基础。
**常见面试问法**：
- "RTOS任务有哪些状态？怎么转换的？"
- "阻塞和挂起有什么区别？"
**回答框架**：
1. 就绪（Ready）：等待CPU调度，条件满足只差执行
2. 运行（Running）：正在使用CPU，同一时刻只有一个
3. 阻塞（Blocked）：等待事件（信号量/消息队列/延时），事件到来转入就绪
4. 挂起（Suspended）：被显式挂起，不参与调度，需显式恢复
5. 阻塞vs挂起：阻塞是被动等待事件，挂起是主动暂停调度
**延伸考点**：任务调度 → 任务间通信

### 任务间通信 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：信号量用于同步和互斥，消息队列用于数据传递，事件标志组用于多条件同步，三者适用场景不同。
**常见面试问法**：
- "信号量和消息队列有什么区别？"
- "互斥信号量和二值信号量有什么区别？"
**回答框架**：
1. 信号量：计数信号量（资源计数）、二值信号量（同步/互斥）、互斥信号量（带优先级继承的互斥）
2. 消息队列：任务间传递数据，支持多条消息缓存，先进先出
3. 事件标志组：一个或多个位的组合，支持"与"/"或"触发，适合多条件等待
4. 选型：互斥用互斥信号量、同步用二值信号量、传数据用消息队列、多条件用事件
**延伸考点**：优先级翻转 → FreeRTOS核心

### FreeRTOS核心概念 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：FreeRTOS是最流行的开源RTOS，核心是任务调度、时间管理、任务间通信，轻量且可裁剪。
**常见面试问法**：
- "FreeRTOS的任务怎么创建？"
- "FreeRTOS的Tick是什么？"
**回答框架**：
1. 任务创建：xTaskCreate()动态创建（堆分配TCB+栈）、xTaskCreateStatic()静态创建（用户分配内存）
2. Tick：系统心跳，configTICK_RATE_HZ配置频率（通常1000Hz），vTaskDelay()基于Tick延时
3. 关键API：vTaskDelayUntil()精确周期延时、vTaskSuspend()/vTaskResume()挂起恢复
4. 内存管理：5种heap方案（heap_1到heap_5），heap_4最常用（合并空闲块），heap_5支持非连续内存
**延伸考点**：任务间通信 → 优先级翻转

### 优先级翻转与优先级继承 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：优先级翻转是高优先级任务被低优先级任务间接阻塞的现象，优先级继承是解决此问题的核心机制。
**常见面试问法**：
- "什么是优先级翻转？怎么解决？"
- "优先级继承的原理是什么？"
**回答框架**：
1. 翻转场景：低优先级任务L持有互斥锁→中优先级任务M抢占L→高优先级任务H等待L释放锁→H被M间接阻塞
2. 优先级继承：H等待L的锁时，L的优先级临时提升到H的级别，防止M抢占L
3. FreeRTOS实现：互斥信号量（xSemaphoreCreateMutex）内置优先级继承，二值信号量没有
4. 其他方案：优先级天花板（将锁的优先级设为所有使用者的最高优先级）
**延伸考点**：FreeRTOS核心 → 任务调度

## 五、Linux嵌入式

### 交叉编译环境 [频率: ⭐⭐⭐⭐]
**核心要点**：交叉编译是在宿主机上生成目标机可执行代码的过程，需配置交叉编译工具链（arm-linux-gnueabihf-gcc等）。
**常见面试问法**：
- "什么是交叉编译？为什么要交叉编译？"
- "交叉编译工具链怎么配置？"
**回答框架**：
1. 为什么交叉编译：目标板资源有限（CPU/内存），无法运行编译器，需在宿主机编译
2. 工具链组成：binutils（汇编/链接）、gcc（编译器）、glibc（C库）、gdb（调试）
3. 配置：设置CC=arm-linux-gnueabihf-gcc，配置--host和--target参数
4. 构建工具：Buildroot（简单）、Yocto（灵活但复杂），用于构建完整根文件系统
**延伸考点**：Bootloader → Linux内核裁剪

### Bootloader（U-Boot） [频率: ⭐⭐⭐⭐⭐]
**核心要点**：U-Boot是最常用的开源Bootloader，负责初始化硬件、加载内核、传递启动参数，是系统启动的第一段代码。
**常见面试问法**：
- "U-Boot的启动流程是什么？"
- "U-Boot怎么加载Linux内核？"
**回答框架**：
1. 启动阶段：SPL（初始化DDR/时钟）→U-Boot主体（初始化外设/网络/存储）
2. 环境变量：bootcmd定义自动启动命令、bootargs定义内核启动参数（root/console/rootfstype）
3. 加载内核：从MMC/NAND/TFTP读取内核镜像到内存→bootm启动
4. 设备树：U-Boot将设备树地址传给内核，内核根据设备树初始化硬件
**延伸考点**：启动流程 → Linux内核裁剪

### Linux内核裁剪与设备树 [频率: ⭐⭐⭐⭐]
**核心要点**：内核裁剪通过menuconfig去除不需要的驱动和功能减小内核体积，设备树用DTS描述硬件信息实现内核与硬件描述解耦。
**常见面试问法**：
- "Linux内核怎么裁剪？"
- "设备树的作用是什么？DTS和DTSO有什么区别？"
**回答框架**：
1. 内核裁剪：make menuconfig配置→去掉不需要的驱动/文件系统/网络协议→减小体积和启动时间
2. 设备树：用DTS文件描述硬件（CPU/内存/外设/中断），编译为DTB二进制，内核解析DTB初始化硬件
3. DTS vs DTSO：DTS是完整设备树，DTSO是设备树覆盖（overlay），动态加载/卸载硬件描述
4. 优势：内核代码与硬件描述分离，同一内核适配不同板子，只需更换DTB
**延伸考点**：U-Boot → 驱动开发

### 驱动开发 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：Linux驱动分为字符设备/块设备/网络设备，字符设备最基础，驱动通过file_operations向用户空间提供接口。
**常见面试问法**：
- "字符设备驱动的开发流程？"
- "驱动怎么和用户空间通信？"
**回答框架**：
1. 字符设备驱动：申请设备号→初始化cdev→实现file_operations（open/read/write/ioctl/release）→创建设备节点
2. file_operations：用户空间open/read/write触发驱动的对应函数
3. 用户空间通信：copy_to_user()（内核→用户）、copy_from_user()（用户→内核）、ioctl自定义命令
4. 设备树匹配：驱动通过of_match_table与设备树节点匹配，probe函数中获取资源
**延伸考点**：Linux内核裁剪 → 文件系统

### 文件系统 [频率: ⭐⭐⭐⭐]
**核心要点**：嵌入式常用JFFS2/YAFFS2（NOR/NAND Flash专用）和ext4（eMMC/SD），选择取决于存储介质和读写需求。
**常见面试问法**：
- "JFFS2和YAFFS2有什么区别？"
- "嵌入式系统怎么选择文件系统？"
**回答框架**：
1. JFFS2：适用于NOR Flash，日志结构、磨损均衡、压缩存储、挂载慢（需扫描全Flash）
2. YAFFS2：适用于NAND Flash，日志结构、磨损均衡、挂载快、不支持压缩
3. ext4：适用于eMMC/SD卡，日志式文件系统、大文件支持好、不适用于裸Flash
4. 选型：NOR Flash用JFFS2、NAND Flash用YAFFS2/UBIFS、eMMC用ext4
**延伸考点**：驱动开发 → Bootloader

## 六、高频项目方向

### RTOS移植 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：FreeRTOS移植到新MCU需修改端口层代码（port.c/portmacro.h），配置时钟和中断优先级，是嵌入式开发的基础能力。
**常见面试问法**：
- "FreeRTOS怎么移植到新平台？"
- "移植过程中遇到过什么问题？"
**回答框架**：
1. 移植步骤：获取FreeRTOS源码→修改port.c（上下文切换/中断处理）→修改portmacro.h（数据类型/内联）→配置FreeRTOSConfig.h→编写启动任务
2. 关键配置：configCPU_CLOCK_HZ（CPU频率）、configTICK_RATE_HZ（Tick频率）、configMAX_SYSCALL_INTERRUPT_PRIORITY（中断优先级阈值）
3. 常见问题：PendSV优先级配置错误、SysTick中断优先级设置、栈溢出
4. 验证：创建闪烁LED任务和串口打印任务，确认调度正常
**延伸考点**：FreeRTOS核心 → 任务调度

### 驱动开发项目 [频率: ⭐⭐⭐⭐]
**核心要点**：传感器驱动/显示驱动是嵌入式高频项目，核心是理解硬件时序、实现内核驱动模块、提供用户空间接口。
**常见面试问法**：
- "你的驱动是怎么写的？"
- "字符设备驱动的框架是什么？"
**回答框架**：
1. 开发流程：阅读芯片数据手册→理解通信时序→编写驱动框架→实现读写操作→调试验证
2. 字符设备框架：模块加载（申请设备号/cdev注册/创建类和设备节点）→file_operations实现→模块卸载
3. 调试方法：dmesg查看内核日志、devmem直接读写寄存器、示波器/逻辑分析仪验证时序
4. 常见传感器：I2C传感器（温湿度/加速度计）、SPI传感器（显示屏/Flash）
**延伸考点**：驱动开发 → 通信协议

### 物联网终端项目 [频率: ⭐⭐⭐⭐]
**核心要点**：物联网终端采集传感器数据通过MQTT上报云平台，核心是传感器驱动+通信模块+MQTT协议+低功耗设计。
**常见面试问法**：
- "物联网终端的架构是什么？"
- "怎么保证数据不丢失？"
**回答框架**：
1. 架构：传感器采集→MCU处理→通信模块（WiFi/4G/NB-IoT）→MQTT→云平台
2. 数据可靠性：QoS1保证至少一次送达、本地Flash缓存断网数据、重连后补传
3. 低功耗：休眠模式+定时唤醒采集、数据批量上报、关闭空闲外设
4. OTA升级：通过MQTT接收固件→写入Flash备份区→校验→重启切换到新固件
**延伸考点**：MQTT → 低功耗设计

## 七、项目高频追问

### RTOS任务调度追问 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：面试官通过追问验证你对RTOS调度机制的深入理解，重点考察PendSV中断、上下文切换和优先级管理。
**常见面试问法**：
- "RTOS任务调度怎么实现的？"
- "上下文切换保存了什么？"
**回答框架**：
1. 调度触发：SysTick中断→检查是否有更高优先级任务就绪→触发PendSV
2. 上下文切换：PendSV中断中保存当前任务寄存器（R4-R11）到任务栈→更新栈指针→恢复新任务寄存器→返回
3. 保存内容：R4-R11（R0-R3/R12/LR/PC/xPSR由硬件自动压栈）、任务栈指针
4. 调度器数据结构：就绪列表（按优先级组织的链表数组）、延时列表
**延伸考点**：任务调度 → ARM中断处理

### 驱动开发追问 [频率: ⭐⭐⭐⭐]
**核心要点**：追问重点考察你对驱动框架的理解深度和实际调试经验，而非简单的API调用。
**常见面试问法**：
- "驱动怎么写的？字符设备框架？"
- "驱动中怎么处理并发？"
**回答框架**：
1. 框架回顾：模块加载→cdev注册→file_operations实现→设备节点创建→模块卸载
2. 并发处理：自旋锁（中断上下文）、互斥锁（进程上下文）、原子操作（简单计数）
3. 中断处理：上半部（硬中断，快速处理）→下半部（软中断/tasklet/工作队列，耗时处理）
4. 调试经验：dmesg内核日志、ftrace函数追踪、逻辑分析仪验证时序
**延伸考点**：驱动开发 → Linux内核

### 中断响应时间 [频率: ⭐⭐⭐⭐]
**核心要点**：中断响应时间是从中断触发到执行ISR第一条指令的时间，受中断延迟、优先级配置和中断屏蔽影响。
**常见面试问法**：
- "中断响应时间多少？"
- "怎么优化中断响应时间？"
**回答框架**：
1. 定义：中断触发→CPU响应→保存现场→跳转ISR，Cortex-M3约12个时钟周期
2. 影响因素：中断是否被屏蔽、是否有同优先级中断在执行、Flash等待周期
3. 优化：提高中断优先级、减少中断嵌套、ISR放RAM执行（零等待）、关闭未使用中断
4. 测量：GPIO翻转+示波器测量，或使用DWT周期计数器精确测量
**延伸考点**：ARM中断处理 → RTOS任务调度

### 低功耗实现 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：低功耗设计从硬件选型、时钟管理、电源模式、软件策略四个层面入手，核心是减少活跃时间和降低活跃时的功耗。
**常见面试问法**：
- "低功耗怎么实现的？"
- "Sleep/Stop/Standby模式有什么区别？"
**回答框架**：
1. 硬件层面：选低功耗MCU、减少外设、优化电路（去掉多余上拉/LED）
2. 时钟管理：降低主频、关闭未使用外设时钟、使用LSI/LSE替代HSE
3. 电源模式：Sleep（CPU停止外设运行）→Stop（所有时钟停止RAM保持）→Standby（仅RTC/唤醒引脚）
4. 软件策略：事件驱动（中断唤醒）替代轮询、批量处理减少唤醒次数、快速执行后立即休眠
**延伸考点**：STM32时钟树 → 物联网终端项目

### 硬件Bug排查 [频率: ⭐⭐⭐⭐⭐]
**核心要点**：硬件Bug排查遵循"先软后硬、先简后繁"原则，用示波器/逻辑分析仪验证信号，用排除法定位问题。
**常见面试问法**：
- "遇到过什么硬件Bug？怎么排查的？"
- "怎么区分软件问题和硬件问题？"
**回答框架**：
1. 排查原则：先确认软件逻辑正确→再查硬件信号→最后查硬件设计
2. 工具：示波器（模拟信号波形）、逻辑分析仪（数字协议解码）、万用表（电压/通断）
3. 常见硬件Bug：焊接虚焊/短路、时钟配置错误、电源纹波过大、信号完整性问题
4. 区分软硬：用已知正常的代码测试→替换硬件对比→测量关键信号→逐步缩小范围
**延伸考点**：通信协议 → 驱动开发
