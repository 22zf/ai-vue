# 手撕代码高频题库

按岗位分类，标注难度和考察频率，提供解题思路框架、易错点与变体题。

---

## 通用高频（所有岗位必会）

### 反转链表 [难度: ⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 迭代法：维护 prev、curr、next 三个指针，逐节点反转指向
2. 每次将 curr.next 指向 prev，然后三指针各前进一步
3. 循环直到 curr 为空，prev 即为新头节点
4. 递归法：递归到尾节点，回溯时逐层反转指向

**易错点**：
- 迭代时必须先保存 next 再修改 curr.next，否则链表断裂
- 递归法返回值是新的头节点，不要误返回原头节点

**变体题**：
- 反转链表 II（指定区间反转）
- K 个一组翻转链表

### 合并两个有序链表 [难度: ⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 使用哑节点（dummy）简化头节点处理
2. 双指针分别遍历两个链表，每次取较小值接入结果链
3. 某链表遍历完后，直接拼接另一链表剩余部分
4. 返回 dummy.next

**易错点**：
- 忘记处理其中一个链表剩余节点
- 未使用哑节点导致头节点插入逻辑复杂化

**变体题**：
- 合并K个升序链表
- 合并两个有序数组

---

### 链表环检测 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 快慢指针法：慢指针走一步，快指针走两步
2. 若存在环，快慢指针必在环内某节点相遇
3. 若快指针到达 null，则无环
4. 找环入口：相遇后，一指针从头部出发，另一指针从相遇点出发，两者再次相遇即为环入口

**易错点**：
- 快指针判断条件需同时检查 fast 和 fast.next 是否为空
- 找环入口的数学推导：相遇点到环入口的距离等于头节点到环入口的距离

**变体题**：
- 返回链表环的入口节点
- 计算环的长度

### 二叉树层序遍历 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 使用队列，根节点入队
2. 每层循环：记录当前队列长度 n，依次出队 n 个节点
3. 每个出队节点的左右子节点依次入队
4. 按层收集节点值，形成二维列表

**易错点**：
- 需在每层开始时固定队列长度，否则会混入下一层节点
- 空树直接返回空列表

**变体题**：
- 二叉树的锯齿形层序遍历
- 二叉树右视图

### 二叉树最大深度 [难度: ⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 递归法：空节点深度为 0
2. 当前节点深度 = max(左子树深度, 右子树深度) + 1
3. BFS 法：层序遍历，记录遍历层数即为最大深度

**易错点**：
- 递归终止条件是节点为 null 返回 0，不是返回 -1
- 注意深度与高度的区别（根节点深度为 1 还是 0 需与面试官确认）

**变体题**：
- 二叉树最小深度
- 判断是否为平衡二叉树

### 有效的括号 [难度: ⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 遇到左括号入栈
2. 遇到右括号，检查栈顶是否为对应左括号，是则出栈，否则无效
3. 遍历结束后栈为空则有效，否则无效

**易错点**：
- 右括号出现时栈可能为空，需先判空
- 只有一种括号匹配不算有效，必须类型对应

**变体题**：
- 最长有效括号子串
- 删除无效括号

### 两数之和 [难度: ⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 遍历数组，对每个元素计算 complement = target - nums[i]
2. 用哈希表记录已遍历的值与下标
3. 若 complement 在哈希表中，直接返回两个下标
4. 否则将当前值加入哈希表

**易错点**：
- 先查找再插入，避免同一元素被重复使用
- 返回的是下标而非值

**变体题**：
- 三数之和
- 四数之和

### 三数之和 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 数组排序，固定第一个数 nums[i]
2. 在 i+1 到末尾区间用双指针 left、right 查找两数之和等于 -nums[i]
3. 根据三数之和与 0 的关系移动指针
4. 找到答案后跳过重复元素，避免重复三元组

**易错点**：
- 必须排序后才能使用双指针
- 去重：固定数和双指针移动后都需跳过重复值

**变体题**：
- 最接近的三数之和
- 四数之和

### 快排/归并排序 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 快排：选取基准值，将小于基准的放左、大于的放右，递归排序左右子数组
2. 归并：递归拆分数组为单元素，再两两合并有序子数组
3. 快排分区用双指针交换法或挖坑填数法
4. 归并合并用双指针比较取较小值

**易错点**：
- 快排分区时指针交错条件容易写错
- 归并排序合并时需用临时数组，不能原地覆盖

**变体题**：
- 排序链表（归并思想）
- 数组中的第K个最大元素（快排分区思想）

### 二分查找 [难度: ⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 初始化 left = 0, right = len - 1
2. 循环条件 left <= right，计算 mid = left + (right - left) / 2
3. 目标值等于 mid 返回；小于 mid 则 right = mid - 1；大于 mid 则 left = mid + 1
4. 未找到返回 -1 或插入位置

**易错点**：
- 循环条件是 <= 还是 < 取决于区间开闭
- mid 计算用 left + (right - left) / 2 防止溢出
- 左右边界更新是 mid ± 1 还是 mid 取决于题目类型

**变体题**：
- 搜索旋转排序数组
- 在排序数组中查找元素的第一个和最后一个位置

### TopK问题 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 堆解法：维护大小为 K 的小顶堆，遍历数组，大于堆顶则替换
2. 快排分区法：随机选基准分区，根据基准位置与 K 的关系缩小范围
3. 堆解法时间 O(nlogK)，快排分区法平均 O(n)
4. 数据量极大或流式数据用堆，内存足够用快排分区

**易错点**：
- 求前 K 大用小顶堆，求前 K 小用大顶堆，容易搞反
- 快排分区法需随机化基准避免最坏情况

**变体题**：
- 数据流中的第K大元素
- 前K个高频元素

### 最小栈 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 维护两个栈：数据栈和辅助栈（最小值栈）
2. 每次入栈时，辅助栈压入当前最小值（新元素与辅助栈顶的较小者）
3. 出栈时两个栈同步弹出
4. 获取最小值直接返回辅助栈顶

**易错点**：
- 辅助栈在入栈第一个元素时需特殊处理（无栈顶可比较）
- 出栈时两个栈必须同步操作

**变体题**：
- 用栈实现队列
- 最大栈

---

### 字符串解码 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 使用两个栈：数字栈和字符串栈
2. 遇到数字时解析完整的多位数字
3. 遇到 [ 时，当前字符串压栈，重置当前字符串
4. 遇到 ] 时，弹出数字和字符串，拼接重复结果

**易错点**：
- 数字可能是多位数，需完整解析而非逐字符处理
- 嵌套场景下字符串拼接顺序容易搞反

**变体题**：
- 基本计算器（含加减括号）
- 压缩字符串

## C++/Java后端特有

### LRU缓存 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 哈希表 + 双向链表，哈希表 O(1) 查找，链表维护访问顺序
2. get 时：查找节点，存在则移到链表头部
3. put 时：存在则更新值并移到头部；不存在则新建节点插入头部
4. 超出容量时删除链表尾部节点（最久未使用）

**易错点**：
- 双向链表插入和删除时需同时修改前后节点的指针，容易漏改
- 删除节点后必须同步删除哈希表中的映射

**变体题**：
- LFU缓存
- 带TTL的LRU缓存

---

### LFU缓存 [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 维护 key→node 映射、freq→双向链表映射、最小访问频次 minFreq
2. get 时：更新节点频次，从旧频次链表移到新频次链表
3. put 时：若存在则更新值和频次；不存在则新建频次为 1 的节点
4. 淘汰时删除 minFreq 对应链表的尾部节点

**易错点**：
- 更新频次后需同步更新 minFreq，尤其是插入新节点时 minFreq 重置为 1
- 同一频次链表中需维护 LRU 顺序作为二级淘汰策略

**变体题**：
- O(1) 时间删除/插入的双向链表节点
- 设计文件系统缓存

### 手写shared_ptr [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 内部维护引用计数指针（int*），多个 shared_ptr 共享同一计数
2. 构造时计数置 1，拷贝构造时计数 +1
3. 析构时计数 -1，计数为 0 时释放资源
4. 需实现构造、拷贝构造、赋值运算符、析构函数

**易错点**：
- 引用计数必须用指针而非整型成员，否则多个 shared_ptr 各自独立计数
- 赋值运算符需先递增右侧计数再递减左侧计数，防止自我赋值时提前释放

**变体题**：
- 手写 unique_ptr
- 手写 weak_ptr 解决循环引用

### 生产者消费者 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 共享缓冲区（队列），生产者放入数据，消费者取出数据
2. 互斥锁保护缓冲区访问
3. 条件变量：缓冲区满时生产者等待，缓冲区空时消费者等待
4. 生产者放入后通知消费者，消费者取出后通知生产者

**易错点**：
- 条件变量的 wait 必须用 while 循环判断，不能用 if（防止虚假唤醒）
- 通知时优先使用 notify_one 而非 notify_all，避免惊群效应

**变体题**：
- 多生产者多消费者
- 有界缓冲区与无界缓冲区的区别

### 线程池 [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 初始化固定数量的工作线程，每个线程循环从任务队列取任务执行
2. 任务队列用互斥锁 + 条件变量保护，队列为空时线程等待
3. submit 接口将任务包装为 std::function 放入队列，并通知一个线程
4. 析构时设置停止标志，通知所有线程退出

**易错点**：
- 析构时需确保所有任务执行完毕再退出，不能强行终止
- 任务队列的互斥锁范围要尽量小，避免持锁时间过长

**变体题**：
- 支持优先级的线程池
- 支持 future 返回值的线程池

### 带TTL的LRU [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 在 LRU 基础上为每个节点增加过期时间戳
2. get 时检查是否过期，过期则删除并返回未命中
3. 定期清理或惰性清理过期节点
4. 可用时间轮或最小堆管理过期时间

**易错点**：
- 过期清理的时机选择：惰性删除简单但可能内存堆积，主动清理需额外线程
- 并发场景下过期检查与删除需保证原子性

**变体题**：
- 实现带过期时间的哈希表
- 时间轮算法实现定时器

### 跳表实现 [难度: ⭐⭐⭐⭐] [频率: ⭐⭐]

**解题思路**：
1. 多层有序链表，底层包含所有节点，上层是下层的稀疏索引
2. 查找：从最高层开始，逐层向下定位目标范围
3. 插入：先查找位置，再以随机层数插入各层
4. 随机层数决定节点在多少层出现，概率通常为 1/2 逐层递减

**易错点**：
- 插入时需从底层到高层逐层更新前驱节点指针
- 随机层数生成需控制最大层数上限，防止极端情况

**变体题**：
- 用跳表实现有序集合（ZSET）
- 跳表与平衡树的性能对比

### 排序链表 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 使用归并排序，递归拆分链表为两半（快慢指针找中点）
2. 对两半分别排序
3. 合并两个有序链表
4. 递归终止条件为链表为空或只有一个节点

**易错点**：
- 拆分时需将前半段尾节点断开（next 置 null），否则无法终止
- 快慢指针找中点时，快指针初始应与慢指针同速出发或快一步

**变体题**：
- 合并K个升序链表
- 重排链表

### 合并K个升序链表 [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 最小堆法：将每个链表的头节点放入最小堆
2. 每次弹出堆顶最小节点接入结果链，并将该节点的 next 压入堆
3. 堆为空时合并完成
4. 分治法：两两合并，递归减半

**易错点**：
- 堆中需记录节点来自哪个链表，以便取 next 继续入堆
- 某链表为空时不应入堆

**变体题**：
- 合并两个有序链表
- K路归并排序

## AI算法特有

### 手写线性回归 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 初始化权重 w 和偏置 b
2. 前向传播计算预测值 y_hat = wx + b
3. 计算损失 MSE = mean((y_hat - y)^2)
4. 反向传播求梯度，梯度下降更新参数
5. 迭代至损失收敛

**易错点**：
- 梯度计算时漏除样本数（批量梯度 vs 随机梯度）
- 学习率过大导致发散，过小导致收敛慢

**变体题**：
- 手写岭回归（L2正则化）
- 手写逻辑回归

### 手写KNN [难度: ⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 存储训练集数据
2. 对新样本计算与所有训练样本的距离（欧氏距离）
3. 选取距离最近的 K 个邻居
4. 投票决定分类（分类任务）或取均值（回归任务）

**易错点**：
- 特征未归一化时，量纲大的特征会主导距离计算
- K 值选择：太小容易过拟合，太大容易欠拟合

**变体题**：
- 加权KNN（距离越近权重越大）
- KD树加速KNN搜索

### 手写注意力机制 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 输入序列经线性变换得到 Q、K、V 三个矩阵
2. 计算注意力分数 scores = Q @ K^T / sqrt(d_k)
3. 对 scores 做 softmax 得到注意力权重
4. 输出 = 权重 @ V

**易错点**：
- 缩放因子 sqrt(d_k) 不能漏，否则梯度消失或爆炸
- softmax 前需做数值稳定性处理（减最大值）

**变体题**：
- 多头注意力机制
- 手写Transformer完整模块

---

### 反向传播推导 [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 前向传播逐层计算输出
2. 计算损失函数对输出层的梯度
3. 利用链式法则逐层反向传递梯度
4. 每层梯度 = 上层梯度 × 本层局部梯度
5. 根据梯度更新参数

**易错点**：
- 矩阵求导时注意维度匹配，转置容易搞错
- 梯度累积与梯度替换的区别（批量训练时需累加）

**变体题**：
- 推导交叉熵损失对 softmax 输入的梯度
- 推导 LSTM 的梯度流

### 手写卷积 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 输入特征图与卷积核逐元素相乘再求和
2. 滑动窗口遍历输入的每个位置
3. 处理多通道：各通道卷积结果相加
4. 可选 padding 和 stride

**易错点**：
- 输出尺寸计算：(input - kernel + 2*padding) / stride + 1
- 多通道卷积时各通道结果需相加而非拼接

**变体题**：
- 手写反卷积（转置卷积）
- 手写深度可分离卷积

### 手写Decision Tree [难度: ⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 选择最优划分特征：计算信息增益（ID3）或信息增益率（C4.5）或基尼系数（CART）
2. 按最优特征划分数据集
3. 递归构建子树
4. 终止条件：节点纯度达标或样本数不足

**易错点**：
- 信息增益偏向取值多的特征，C4.5 用增益率修正
- 连续值特征需寻找最佳分裂点

**变体题**：
- 手写随机森林
- 手写GBDT

### 手写Transformer [难度: ⭐⭐⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 词嵌入 + 位置编码
2. 编码器：多头注意力 → 残差+LayerNorm → FFN → 残差+LayerNorm
3. 解码器：掩码多头注意力 → 交叉注意力 → FFN，各层残差+LayerNorm
4. 输出经线性层 + softmax 得到概率分布

**易错点**：
- 解码器的掩码注意力需屏蔽未来位置
- 位置编码使用正弦余弦公式，不是可学习参数（原始论文）

**变体题**：
- 手写GPT（仅解码器）
- 手写BERT（仅编码器）

## 前端特有

### 防抖debounce [难度: ⭐⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 维护一个定时器 ID
2. 每次调用时清除上一次定时器，重新设置新定时器
3. 定时器到期后执行目标函数
4. 返回包装后的函数

**易错点**：
- this 指向和参数传递需用 apply/call 或箭头函数处理
- 需要提供取消方法（cancel）和立即执行方法（flush）

**变体题**：
- 带立即执行选项的防抖
- 防抖与节流的区别及适用场景

---

### 节流throttle [难度: ⭐⭐] [频率: ⭐⭐⭐⭐⭐]

**解题思路**：
1. 时间戳法：记录上次执行时间，当前时间差大于间隔则执行
2. 定时器法：设置定时器，到期执行并清空
3. 时间戳+定时器结合：确保首次立即执行且最后一次也执行
4. 返回包装后的函数

**易错点**：
- 纯时间戳法最后一次触发可能不执行
- 纯定时器法首次触发会延迟

**变体题**：
- 带尾部执行的节流
- 节流函数的取消功能

---

### 深拷贝 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 基本类型直接返回
2. 对象/数组递归拷贝每个属性
3. 用 WeakMap 记录已拷贝对象，处理循环引用
4. 特殊对象（Date、RegExp、Map、Set 等）需单独处理

**易错点**：
- 循环引用会导致无限递归，必须用 WeakMap 缓存
- 函数、Symbol、DOM 节点等特殊类型的处理策略需与面试官确认

**变体题**：
- 结构化克隆（structuredClone）
- 实现一个简易的 JSON.stringify

---

### Promise.all [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 返回新 Promise，维护计数器和结果数组
2. 遍历所有 Promise，每个 resolve 时计数器 +1 并存储结果
3. 计数器等于 Promise 总数时，外层 Promise resolve
4. 任一 Promise reject 时，外层 Promise 立即 reject

**易错点**：
- 传入非 Promise 值需用 Promise.resolve 包装
- 结果数组需按原始顺序存储，不能按完成顺序

**变体题**：
- Promise.allSettled
- Promise.race / Promise.any

---

### call/apply/bind [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. call：将函数设为对象的属性，执行后删除该属性
2. apply：与 call 类似，参数以数组形式传入
3. bind：返回新函数，新函数执行时 this 绑定到指定对象
4. bind 返回的函数支持后续传参（偏函数）

**易错点**：
- bind 返回的函数作为构造函数时，this 应指向实例而非绑定的对象
- null/undefined 作为 this 时需指向全局对象（非严格模式）

**变体题**：
- 实现软绑定（softBind）
- 实现 Function.prototype.call 的 polyfill

---

### 发布订阅模式 [难度: ⭐⭐] [频率: ⭐⭐⭐⭐]

**解题思路**：
1. 维护事件中心对象，key 为事件名，value 为回调函数数组
2. on：注册事件，将回调推入对应数组
3. emit：触发事件，依次执行对应数组中的所有回调
4. off：取消订阅，从数组中移除指定回调

**易错点**：
- off 时需比较函数引用，匿名函数无法正确移除
- emit 时需拷贝回调列表再遍历，防止回调中修改列表

**变体题**：
- 支持一次性的 once 方法
- 观察者模式与发布订阅模式的区别

---

### 数组扁平化 [难度: ⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 递归法：遍历数组，元素为数组则递归展开，否则直接收集
2. reduce 法：reduce 累加，遇到数组元素递归调用 flatten
3. 迭代法：用栈，弹出元素是数组则展开压回栈中
4. 指定展开深度参数控制递归层数

**易错点**：
- 需支持指定展开深度，深度为 0 时不展开
- 空数组元素需保留还是跳过需确认

**变体题**：
- Array.prototype.flat 的 polyfill
- 嵌套数组的求和

---

### 虚拟DOM Diff [难度: ⭐⭐⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 同层比较：只比较同一层级的节点
2. 类型不同直接替换，类型相同则更新属性
3. 列表比较使用 key 优化：相同 key 复用节点，不同 key 新建
4. 生成差异补丁（patch），批量应用到真实 DOM

**易错点**：
- 不使用 key 时采用首尾指针法，可能产生不必要的移动
- key 应使用稳定唯一标识，index 作 key 等于没有 key

**变体题**：
- 手写简易 Vue 响应式系统
- 最长递增子序列在 Diff 中的应用

### 排序算法 [难度: ⭐⭐] [频率: ⭐⭐⭐]

**解题思路**：
1. 掌握常见排序：冒泡、选择、插入、快排、归并、堆排
2. 关注时间复杂度、空间复杂度、稳定性
3. 稳定排序：冒泡、插入、归并；不稳定排序：选择、快排、堆排

**易错点**：

**变体题**：
- 对几乎有序的数组选择哪种排序
- 稳定排序与不稳定排序的实际影响
