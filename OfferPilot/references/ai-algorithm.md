### 线性回归与正则化 [频率: ⭐⭐⭐⭐]

**核心要点**：线性回归通过最小化残差平方和拟合超平面，L1正则化产生稀疏解（Lasso），L2正则化收缩权重（Ridge），ElasticNet结合两者优势。

**常见面试问法**：
- "L1正则化为什么能产生稀疏解？L2为什么不能？"
- "Ridge回归和Lasso回归分别适用于什么场景？"

**回答框架**：
1. 先说线性回归的目标函数：min ||y - Xw||²
2. 引入正则化：L1为|w|₁，L2为||w||₂²，从贝叶斯视角L1对应Laplace先验，L2对应Gaussian先验
3. 解释L1稀疏性：L1在零点处不可导，等高线与菱形约束易在顶点相交；L2等高线与圆形约束不易在轴上相交
4. 应用场景：Lasso用于特征选择，Ridge用于多重共线性，ElasticNet用于高维且特征相关场景

**延伸考点**：偏差方差权衡 → 正则化 → 特征选择方法

### 逻辑回归 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：逻辑回归是线性分类模型，用sigmoid函数将线性输出映射到概率，通过极大似然估计求解，本质是对数线性模型。

**常见面试问法**：
- "逻辑回归为什么用对数损失而不用平方损失？"
- "逻辑回归怎么做多分类？"

**回答框架**：
1. 定义：P(y=1|x) = σ(wᵀx)，σ(z) = 1/(1+e⁻ᶻ)
2. 损失函数：交叉熵损失 L = -[ylogŷ + (1-y)log(1-ŷ)]，凸函数有全局最优解；平方损失非凸，sigmoid导致梯度饱和
3. 求解：梯度下降法，梯度为 Xᵀ(ŷ - y)，形式与线性回归相似
4. 多分类扩展：OvR（一对多）、Softmax回归（多项逻辑回归）
5. 应用场景：CTR预估基线模型、金融风控评分卡

**延伸考点**：sigmoid函数性质 → 交叉熵损失 → 最大熵模型

### SVM与核函数 [频率: ⭐⭐⭐⭐]

**核心要点**：SVM寻找最大间隔超平面，通过核技巧将低维非线性问题映射到高维线性可分，软间隔通过松弛变量处理噪声。

**常见面试问法**：
- "SVM为什么要求对偶问题？核函数的作用是什么？"
- "RBF核、线性核、多项式核怎么选？"

**回答框架**：
1. 原始问题：min ½||w||² s.t. yᵢ(wᵀxᵢ+b)≥1，几何间隔最大化
2. 对偶问题：引入拉格朗日乘子α，对偶问题只涉及内积⟨xᵢ,xⱼ⟩，自然引入核函数K(xᵢ,xⱼ)=φ(xᵢ)ᵀφ(xⱼ)
3. 软间隔：引入松弛变量ξᵢ和惩罚参数C，平衡间隔最大化与分类错误
4. 核函数选择：线性核（高维稀疏特征）、RBF核（通用默认选择，需调γ）、多项式核（特定结构数据）
5. SMO算法求解对偶问题

**延伸考点**：对偶理论 → 核方法 → Hinge Loss → SVM与逻辑回归对比

### 决策树三种划分标准 [频率: ⭐⭐⭐⭐]

**核心要点**：ID3用信息增益（偏好多值特征）、C4.5用信息增益率（克服多值偏好）、CART用基尼指数（二叉划分，回归用平方误差）。

**常见面试问法**：
- "信息增益和信息增益率有什么区别？为什么C4.5用增益率？"
- "CART怎么做回归？"

**回答框架**：
1. 信息增益：Gain(D,a) = Ent(D) - Σ|Dᵛ|/|D|·Ent(Dᵛ)，Ent为信息熵，偏好取值多的特征
2. 信息增益率：Gain_ratio(D,a) = Gain(D,a) / IV(a)，IV(a)为固有值，惩罚多值特征，但可能偏好取值少的特征，C4.5先筛增益高于平均再选增益率最大
3. 基尼指数：Gini(D) = 1 - Σpₖ²，衡量数据纯度，CART每次选最小基尼指数的特征及切分点做二叉划分
4. CART回归：遍历特征和切分点，选使平方误差最小的划分，叶节点输出为均值
5. 对比总结：ID3只能分类且偏好多值，C4.5克服偏好可处理连续值，CART二叉树可分类可回归

**延伸考点**：信息熵 → 随机森林 → GBDT → 过拟合与剪枝

### 随机森林 [频率: ⭐⭐⭐]

**核心要点**：随机森林基于Bagging集成多棵决策树，引入特征随机选择降低相关性，袋外误差（OOB）可作为无偏估计。

**常见面试问法**：
- "随机森林为什么能降低方差？特征随机选择的作用是什么？"
- "袋外误差怎么计算？有什么优势？"

**回答框架**：
1. Bagging原理：对训练集有放回采样，训练多棵树，投票/平均输出
2. 特征随机：每棵树每个节点从√p（分类）或p/3（回归）个特征中选最优分裂，降低树间相关性
3. 方差降低：Var(平均) = ρσ²/n + (1-ρ)σ²/n，相关性ρ越小方差降低越多
4. OOB：每棵树约63.2%样本参与训练，未参与样本用于评估，无需额外验证集
5. 优势：抗过拟合、可并行、可特征重要性评估

**延伸考点**：Bagging vs Boosting → 特征重要性 → XGBoost

### XGBoost vs LightGBM vs CatBoost [频率: ⭐⭐⭐⭐⭐]

**核心要点**：XGBoost用预排序+近似算法，LightGBM用直方图加速+Leaf-wise生长，CatBoost用Ordered Boosting处理类别特征，三者各有适用场景。

**常见面试问法**：
- "XGBoost和LightGBM的核心区别是什么？为什么LightGBM更快？"
- "CatBoost怎么处理类别特征的？有什么优势？"

**回答框架**：
1. XGBoost：预排序精确贪心/近似算法，Level-wise层序生长，正则化(γ,λ,α)，支持缺失值自动学习分裂方向
2. LightGBM：直方图算法将连续值离散化为bin，降低内存和计算量；Leaf-wise按增益选叶分裂，深度可能过大需限制max_depth；GOSS保留大梯度样本随机丢小梯度；EFB合并互斥稀疏特征
3. CatBoost：Ordered Boosting避免目标泄露，类别特征用目标统计量编码（Ordered TS），对称树结构加速推理
4. 对比：XGBoost精度稳定但慢，LightGBM快但小数据易过拟合，CatBoost类别特征强且无需手动编码
5. 选择建议：大数据选LightGBM，类别特征多选CatBoost，小数据或追求稳定选XGBoost

**延伸考点**：GBDT原理 → 直方图算法 → 类别特征编码 → 树模型vs神经网络

### KNN [频率: ⭐⭐⭐]

**核心要点**：KNN是惰性学习，通过距离度量找K个最近邻投票分类，K值选择和距离度量是关键，KD树加速搜索。

**常见面试问法**：
- "KNN的K值怎么选？太大太小会怎样？"
- "KD树什么情况下加速效果不好？"

**回答框架**：
1. 原理：对测试样本计算与所有训练样本的距离，取最近K个的多数类为预测
2. K值选择：K太小噪声敏感（过拟合），K太大决策边界过于平滑（欠拟合），通常交叉验证选K，奇数避免平票
3. 距离度量：欧氏距离（连续）、曼哈顿距离、余弦相似度（文本），需特征归一化
4. KD树：对训练数据建KD树，最近邻搜索从O(n)降到O(logn)，但高维时回溯频繁退化为O(n)
5. 优缺点：无需训练、可多分类；预测慢、对不相关特征敏感、高维灾难

**延伸考点**：距离度量 → 维度灾难 → 近似最近邻（ANN） → 向量检索

### 偏差方差权衡 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：模型总误差 = 偏差² + 方差 + 不可约误差，偏差衡量模型拟合能力，方差衡量模型对数据扰动的敏感度，复杂模型低偏差高方差，简单模型高偏差低方差。

**常见面试问法**：
- "什么是偏差方差权衡？怎么判断模型是过拟合还是欠拟合？"
- "Bagging和Boosting分别怎么影响偏差和方差？"

**回答框架**：
1. 定义：偏差 = E[ŷ] - y（预测均值与真实值的偏离），方差 = E[(ŷ - E[ŷ])²]（预测的波动）
2. 权衡：模型复杂度↑ → 偏差↓方差↑（过拟合），复杂度↓ → 偏差↑方差↓（欠拟合）
3. 判断：训练误差高测试误差高→高偏差（欠拟合），训练误差低测试误差高→高方差（过拟合）
4. Bagging降方差（多模型平均），Boosting降偏差（逐步修正残差）
5. 实践策略：欠拟合→增加特征/模型复杂度，过拟合→正则化/增加数据/简化模型

**延伸考点**：过拟合/欠拟合 → 正则化 → 集成学习 → 学习曲线

### 评估指标 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：分类用Accuracy/Precision/Recall/F1/AUC-ROC，回归用MAE/MSE/RMSE/R²，需根据业务场景选择，如医疗重Recall、搜索重Precision。

**常见面试问法**：
- "AUC-ROC的物理含义是什么？为什么适用于不平衡数据？"
- "Precision和Recall怎么权衡？F1是什么？"

**回答框架**：
1. 混淆矩阵：TP/FP/FN/TN，由此推导Precision=TP/(TP+FP)、Recall=TP/(TP+FN)
2. F1 = 2·P·R/(P+R)，调和平均，兼顾P和R；Fβ可调权重
3. AUC-ROC：ROC曲线横轴FPR、纵轴TPR，AUC为曲线下面积，物理含义为随机正样本得分高于随机负样本的概率
4. AUC优势：不依赖阈值选择，对类别不平衡鲁棒（只看排序不看绝对值）
5. 回归指标：MAE对异常值鲁棒，MSE惩罚大误差，R²=1-SSE/SST衡量解释方差比例

**延伸考点**：PR曲线 vs ROC曲线 → 类别不平衡处理 → 代价敏感学习

---

### 交叉验证 [频率: ⭐⭐⭐⭐]

**核心要点**：交叉验证通过多次划分训练集和验证集来更稳健地评估模型，K折是最常用方式，分层K折保证类别分布一致。

**常见面试问法**：
- "K折交叉验证的K怎么选？K太大太小有什么问题？"
- "时间序列数据怎么做交叉验证？"

**回答框架**：
1. K折CV：将数据分成K份，轮流用1份做验证其余做训练，K次结果平均
2. K的选择：K=5或10是经验最优，K太小估计偏差大，K太大方差大且计算量大
3. 留一法（LOO）：K=N，无偏但方差大、计算量大
4. 分层K折：每折中各类别比例与整体一致，适用于不平衡数据
5. 时间序列：前向链式验证（rolling forecast），只能用过去数据预测未来，不能随机打乱

**延伸考点**：模型选择 → 超参调优（Grid/Random/Bayesian） → 数据泄露

### 正则化 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：正则化通过约束模型复杂度防止过拟合，L1产生稀疏（特征选择），L2收缩权重（平滑），Dropout是深度学习中的正则化，Early Stopping是训练时正则化。

**常见面试问法**：
- "L1和L2正则化分别从什么角度防止过拟合？"
- "除了L1/L2，还有哪些正则化方法？"

**回答框架**：
1. L1正则化：损失+λ|w|₁，贝叶斯视角等价于Laplace先验，使部分权重精确为零，实现特征选择
2. L2正则化：损失+λ||w||₂²，贝叶斯视角等价于Gaussian先验，权重趋近于零但不为零，限制权重幅度
3. ElasticNet：α·L1 + (1-α)·L2，兼顾稀疏与平滑
4. 深度学习正则化：Dropout（训练时随机失活）、DropPath（随机丢弃整条路径）、权重衰减（L2的SGD等价形式）、数据增强、Early Stopping
5. 正则化强度λ的选择：交叉验证确定，λ太大欠拟合，λ太小过拟合

**延伸考点**：贝叶斯先验 → 偏差方差 → Dropout → 数据增强

### 优化算法 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：SGD是基础随机梯度下降，Momentum加速收敛，Adam结合一阶矩和二阶矩自适应调整学习率，是深度学习最常用的优化器。

**常见面试问法**：
- "Adam和SGD的区别是什么？为什么Adam收敛快但泛化可能不如SGD？"
- "学习率衰减策略有哪些？Warmup的作用是什么？"

**回答框架**：
1. SGD：θ = θ - η·∇L，简单但震荡，收敛慢
2. Momentum：引入动量v = βv + ∇L，θ = θ - ηv，加速收敛减少震荡
3. AdaGrad：累积梯度平方，自适应降低高频特征学习率，但分母单调增导致学习率趋零
4. RMSprop：指数移动平均替代累积，解决AdaGrad学习率衰减问题
5. Adam：m = β₁m + (1-β₁)∇L（一阶矩），v = β₂v + (1-β₂)∇L²（二阶矩），偏差校正后自适应学习率，β₁=0.9, β₂=0.999
6. 学习率调度：Step/Exponential/Cosine Annealing/OneCycleLR；Warmup在训练初期用小学习率避免不稳定
7. Adam vs SGD：Adam收敛快适合大部分场景，SGD+Momentum泛化更好但需精调学习率

**延伸考点**：梯度消失/爆炸 → 学习率调度 → 梯度裁剪 → 二阶优化方法

### 反向传播与链式法则 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：反向传播利用链式法则逐层计算损失函数对参数的梯度，从输出层向输入层反向传播误差信号，是神经网络训练的核心算法。

**常见面试问法**：
- "请推导反向传播的过程，链式法则怎么用的？"
- "梯度消失和梯度爆炸的原因和解决方案？"

**回答框架**：
1. 前向传播：逐层计算 z = Wx + b, a = σ(z)，保存中间变量
2. 反向传播：∂L/∂W = ∂L/∂a · ∂a/∂z · ∂z/∂W，链式法则逐层展开
3. 梯度消失：sigmoid/tanh导数最大0.25/1，多层连乘梯度指数衰减；解决方案：ReLU、残差连接、BatchNorm、梯度裁剪
4. 梯度爆炸：权重初始化过大导致梯度指数增长；解决方案：梯度裁剪、权重初始化、BatchNorm
5. 计算图视角：自动微分构建计算图，前向记录操作，反向沿图反向传播梯度

**延伸考点**：自动微分 → 残差连接 → 梯度裁剪 → 混合精度训练

---

### 激活函数对比 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：ReLU解决梯度消失且计算高效但有Dying ReLU问题，GELU被Transformer广泛使用，SwiGLU是LLaMA等LLM的选择。

**常见面试问法**：
- "ReLU有什么缺点？LeakyReLU怎么解决的？"
- "GELU和ReLU的区别？为什么Transformer用GELU？"

**回答框架**：
1. Sigmoid：σ(x)=1/(1+e⁻ˣ)，输出(0,1)，梯度最大0.25，易梯度消失，输出非零中心
2. Tanh：输出(-1,1)，零中心，梯度最大1，仍易梯度消失
3. ReLU：max(0,x)，计算快，正区间梯度恒为1，但负区间梯度为零（Dying ReLU），输出非零中心
4. LeakyReLU：x>0时x，x<0时αx（α=0.01），解决Dying ReLU
5. GELU：x·Φ(x)，Φ为标准正态CDF，平滑近似0.5x(1+tanh(…))，概率性门控，Transformer标配
6. SwiGLU：Swish(x)⊗(Wx+b)，Swish=x·σ(βx)，LLaMA/PaLM使用，GLU门控增强表达能力

**延伸考点**：梯度消失/爆炸 → 网络初始化 → Transformer架构

---

### BatchNorm vs LayerNorm [频率: ⭐⭐⭐⭐⭐]

**核心要点**：BatchNorm沿batch维度归一化（CV常用），LayerNorm沿特征维度归一化（NLP/Transformer常用），核心区别在于归一化维度和对序列长度的适应性。

**常见面试问法**：
- "Transformer为什么用LayerNorm不用BatchNorm？"
- "BatchNorm训练和推理的区别是什么？"

**回答框架**：
1. BatchNorm：对每个特征，在batch维度上计算μ和σ，归一化后γβ缩放偏移；训练时用当前batch统计量，推理时用running mean/var
2. LayerNorm：对每个样本，在特征维度上计算μ和σ，不依赖batch，训练推理一致
3. Transformer用LN的原因：序列长度可变，batch中padding导致BN统计量不准；自注意力输出与序列位置相关，BN破坏位置信息；小batch时BN统计量不稳定
4. RMSNorm：LayerNorm的简化版，只除以RMS不做均值中心化，LLaMA使用，计算更快
5. DeepNorm：微软提出，配合残差缩放实现深层Transformer稳定训练

**延伸考点**：归一化 → 残差连接 → Transformer → RMSNorm

---

### Dropout [频率: ⭐⭐⭐⭐]

**核心要点**：Dropout训练时随机将部分神经元输出置零，等效于集成多个子网络，推理时不dropout但缩放输出，是深度学习核心正则化手段。

**常见面试问法**：
- "Dropout训练和推理有什么区别？为什么要缩放？"
- "DropPath和Dropout有什么区别？"

**回答框架**：
1. 训练：每个神经元以概率p被置零，剩余神经元乘1/(1-p)（inverted dropout），推理不dropout
2. 缩放原因：训练时部分神经元失活，推理时全部激活，不缩放则推理输出期望偏大
3. 等效集成：每次dropout产生不同子网络，近似模型集成效果
4. 变体：SpatialDropout（整通道dropout，CNN用）、DropPath（随机丢弃整条残差路径，Transformer用）、DropConnect（丢弃权重而非输出）
5. p的选择：全连接层0.5，CNN的0.1-0.3，Transformer注意力0.1

**延伸考点**：正则化 → 模型集成 → Transformer训练技巧

### 初始化方法 [频率: ⭐⭐⭐⭐]

**核心要点**：好的初始化使各层输出方差稳定，避免梯度消失/爆炸，Xavier适用于tanh/sigmoid，He初始化适用于ReLU及其变体。

**常见面试问法**：
- "为什么不能全零初始化？Xavier和He初始化的区别是什么？"
- "全1初始化或全0初始化会有什么问题？"

**回答框架**：
1. 全零初始化问题：同一层所有神经元梯度相同，对称性无法打破，永远学不到不同特征
2. Xavier初始化：W ~ N(0, 2/(fan_in+fan_out))，保证前向和反向传播时方差不变，适用于tanh/sigmoid
3. He初始化：W ~ N(0, 2/fan_in)，考虑ReLU让一半神经元失活，方差需额外×2补偿，适用于ReLU
4. 其他方法：正交初始化（保持梯度范数）、预训练初始化（迁移学习）
5. 偏置初始化：通常初始化为0，BatchNorm的β初始化为0、γ初始化为1

**延伸考点**：梯度消失/爆炸 → 残差连接 → 迁移学习

### 卷积计算 [频率: ⭐⭐⭐⭐]

**核心要点**：卷积输出尺寸 = (输入尺寸 - 卷积核 + 2×填充) / 步长 + 1，感受野随层数递增，空洞卷积扩大感受野不增加参数。

**常见面试问法**：
- "给定输入尺寸、卷积核大小、步长、填充，计算输出尺寸"
- "感受野怎么计算？空洞卷积有什么用？"

**回答框架**：
1. 输出尺寸公式：O = (I - K + 2P) / S + 1，I输入、K卷积核、P填充、S步长
2. 参数量：K²×C_in×C_out + C_out（偏置）
3. 计算量(FLOPs)：O²×K²×C_in×C_out
4. 感受野：第l层感受野 RF_l = RF_{l-1} + (K_l - 1) × Π_{i=1}^{l-1} S_i
5. 空洞卷积：在卷积核元素间插入dilation-1个零，等效卷积核K_eff = K + (K-1)(dilation-1)，扩大感受野不增参数
6. 深度可分离卷积：Depthwise(每通道独立卷积) + Pointwise(1×1卷积通道融合)，参数量降为1/K²+1/C_out

**延伸考点**：感受野 → 轻量化网络 → 目标检测

### 经典网络ResNet等 [频率: ⭐⭐⭐⭐]

**核心要点**：ResNet通过残差连接解决深层网络退化问题，DenseNet密集连接实现特征复用，EfficientNet用NAS+复合缩放搜索最优结构。

**常见面试问法**：
- "ResNet为什么能训练很深的网络？残差连接的原理是什么？"
- "ResNet为什么用1×1卷积？"

**回答框架**：
1. VGG：3×3小卷积堆叠替代大卷积，参数量大(138M)，证明深度重要性
2. ResNet：残差块 F(x) = H(x) - x，学习残差比学习恒等映射容易；梯度直通路径解决退化；Bottleneck用1×1降维→3×3卷积→1×1升维减少计算量
3. DenseNet：每层接收前面所有层的输出作为输入，特征复用，参数少但显存消耗大
4. EfficientNet：复合缩放(深度/宽度/分辨率同时缩放)，NAS搜索基线结构，用更少参数达到SOTA
5. 残差连接本质：信息高速公路，梯度可以直接回传，缓解梯度消失，使网络可以学习恒等映射

**延伸考点**：梯度消失 → 网络架构搜索 → 迁移学习 → 目标检测Backbone

### 目标检测YOLO/Faster R-CNN [频率: ⭐⭐⭐⭐]

**核心要点**：Faster R-CNN是两阶段检测器（RPN+RoI Pooling），YOLO是单阶段检测器（端到端回归），YOLOv8引入Anchor-Free和Decoupled Head。

**常见面试问法**：
- "两阶段和单阶段检测器的区别？各有什么优缺点？"
- "Faster R-CNN的RPN是怎么工作的？"

**回答框架**：
1. 两阶段（Faster R-CNN）：Backbone→RPN生成候选框→RoI Pooling提取特征→分类+回归；精度高但速度慢
2. RPN：在特征图上滑动，每个位置生成k个Anchor，判断前景/背景，回归偏移量
3. 单阶段（YOLO）：将检测视为回归问题，网格划分直接预测类别和边界框；速度快但小目标精度低
4. YOLO演进：YOLOv1(网格划分)→v3(多尺度FPN)→v5(数据增强)→v8(Anchor-Free, Decoupled Head, Distribution Focal Loss)
5. 关键技术：NMS去重、FPN多尺度特征、Anchor设计、Focal Loss解决正负样本不平衡

**延伸考点**：Anchor机制 → FPN → NMS → 实例分割

---

### 语义分割 [频率: ⭐⭐⭐]

**核心要点**：语义分割是像素级分类任务，FCN开创端到端分割，U-Net对称编码解码+跳跃连接适合医学图像，DeepLab用空洞空间卷积池化金字塔(ASPP)处理多尺度。

**常见面试问法**：
- "U-Net的跳跃连接有什么作用？和ResNet的跳跃连接有什么区别？"
- "DeepLab的ASPP是怎么处理多尺度问题的？"

**回答框架**：
1. FCN：全卷积网络，用1×1卷积替代全连接，转置卷积上采样恢复分辨率，跳跃连接融合多层特征
2. U-Net：编码器下采样提取特征，解码器上采样恢复分辨率，跳跃连接将编码器特征拼接到解码器，保留细节信息；医学图像分割标配
3. DeepLabv3+：ASPP用不同膨胀率的空洞卷积并行提取多尺度特征，编码器-解码器结构恢复边界细节
4. 跳跃连接对比：ResNet是加法（特征融合），U-Net是拼接（特征保留）
5. 评估指标：mIoU（各类IoU平均）、像素准确率、Dice系数

**延伸考点**：实例分割(Mask R-CNN) → 全景分割 → 医学图像分割

### Self-Attention Q/K/V [频率: ⭐⭐⭐⭐⭐]

**核心要点**：Self-Attention通过Q/K/V三个线性变换计算注意力权重，Attention(Q,K,V) = softmax(QKᵀ/√dₖ)V，缩放因子√dₖ防止点积过大导致softmax梯度消失。

**常见面试问法**：
- "Self-Attention的计算流程是什么？为什么要除以√dₖ？"
- "Multi-Head Attention相比单头有什么优势？"

**回答框架**：
1. Q/K/V生成：输入X分别乘W_Q/W_K/W_V得到Query/Key/Value
2. 注意力计算：score = QKᵀ/√dₖ，weight = softmax(score)，output = weight·V
3. 缩放原因：dₖ大时QKᵀ方差大，softmax输入值域大导致输出趋近one-hot，梯度趋近零；除以√dₖ使方差归一化
4. Multi-Head：将d_model拆成h个头，各头独立计算注意力再拼接线性变换，不同头关注不同子空间信息
5. 复杂度：O(n²d)，n为序列长度，这是Transformer处理长序列的瓶颈

**延伸考点**：注意力机制变体 → KV Cache → 稀疏注意力 → Flash Attention

---

### 位置编码RoPE/ALiBi [频率: ⭐⭐⭐⭐⭐]

**核心要点**：Transformer本身无位置感知，位置编码注入序列顺序信息，RoPE通过旋转矩阵实现相对位置编码，ALiBi直接在注意力分数上加距离惩罚。

**常见面试问法**：
- "RoPE的原理是什么？为什么能外推？"
- "ALiBi和RoPE的区别？各有什么优缺点？"

**回答框架**：
1. 正弦位置编码：PE(pos,2i) = sin(pos/10000^{2i/d})，固定不可学习，有一定相对位置信息
2. 可学习位置编码：BERT使用，位置嵌入作为参数学习，不能外推到训练长度之外
3. RoPE（旋转位置编码）：将Q/K视为二维向量对，乘旋转矩阵e^{imθ}，内积自然包含相对位置m-n；远程衰减特性；LLaMA/Qwen使用
4. RoPE外推问题：训练长度外注意力值急剧下降，解决方案：NTK-aware缩放、YaRN、动态NTK
5. ALiBi：在注意力分数上直接加 -m·|i-j| 线性衰减惩罚，m为head-specific斜率，无需位置编码嵌入，外推能力强，BLOOM/MPT使用

**延伸考点**：长度外推 → NTK-aware缩放 → 注意力机制

### LLM GPT/LLaMA/Qwen/DeepSeek [频率: ⭐⭐⭐⭐⭐]

**核心要点**：GPT系列开创自回归大语言模型范式，LLaMA证明小模型高质量数据可匹敌大模型，Qwen是多语言优化，DeepSeek以MoE和低成本训练著称。

**常见面试问法**：
- "GPT系列的发展脉络是什么？GPT-3到GPT-4的关键改进？"
- "LLaMA为什么小模型也能有好的效果？DeepSeek的MoE架构有什么特点？"

**回答框架**：
1. GPT系列：GPT-1(预训练+微调)→GPT-2(零样本)→GPT-3(少样本/In-context Learning, 175B)→GPT-4(多模态, RLHF对齐)
2. LLaMA：7B/13B/33B/65B，1.4T tokens训练，高质量数据（过滤+去重），RoPE+SwiGLU+RMSNorm，证明数据质量>模型规模
3. Qwen：多语言优化（中英为主），RoPE+SwiGLU，Qwen2引入GQA减少KV Cache，支持长上下文
4. DeepSeek：DeepSeek-V2用MLA（Multi-head Latent Attention）压缩KV Cache，DeepSeekMoE细粒度专家+共享专家，训练成本极低
5. 关键技术趋势：GQA/MLA降低推理成本、MoE提升参数效率、长上下文扩展、多模态融合

**延伸考点**：MoE架构 → 长上下文 → RLHF对齐 → 推理优化

### 训练技术SFT/RLHF/DPO [频率: ⭐⭐⭐⭐⭐]

**核心要点**：LLM训练三阶段：预训练（大规模无监督）→SFT（监督微调）→对齐（RLHF/DPO），DPO用直接偏好优化替代RLHF的复杂奖励模型训练。

**常见面试问法**：
- "RLHF的流程是什么？DPO相比RLHF有什么优势？"
- "SFT的数据怎么构造？需要注意什么？"

**回答框架**：
1. 预训练：自回归下一token预测，海量文本，学习语言知识和世界知识
2. SFT：用指令-回答对微调，让模型学会遵循指令；数据质量>数量，需多样性、准确性、格式统一
3. RLHF：①训练奖励模型（人类偏好排序）→②用PPO优化策略模型最大化奖励+KL散度约束；问题：训练不稳定、奖励模型偏差、PPO超参敏感
4. DPO：将RLHF的约束优化转化为直接偏好优化，损失函数 L = -log σ(β(log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))，无需奖励模型，训练稳定
5. 其他对齐方法：RLAIF（AI反馈替代人类）、ORPO（无需参考模型）、KTO（只需好/坏标签不需配对）

**延伸考点**：奖励模型 → PPO算法 → 偏好数据构造 → 模型安全对齐

### 推理优化KV Cache/量化/Flash Attention/vLLM [频率: ⭐⭐⭐⭐⭐]

**核心要点**：KV Cache缓存已计算的K/V避免重复计算，量化降低模型精度减少显存，Flash Attention用分块计算减少HBM访问，vLLM是高效推理服务框架。

**常见面试问法**：
- "KV Cache的原理是什么？GQA怎么减少KV Cache的？"
- "Flash Attention为什么能加速？量化的INT8和INT4有什么区别？"

**回答框架**：
1. KV Cache：自回归生成时缓存已计算的K/V，每步只需计算新token的Q与所有K/V的注意力，显存占用O(n·d·layers)
2. GQA（Grouped Query Attention）：多个Q头共享一组K/V头，LLaMA2/Qwen2使用，减少KV Cache至1/G
3. 量化：PTQ（训练后量化）和QAT（量化感知训练）；INT8权重+激活量化几乎无损，INT4需GPTQ/AWQ等算法保持精度；KV Cache也可量化
4. Flash Attention：核心是分块计算(tiling)，将Q/K/V分块在SRAM中完成注意力计算，减少对HBM的读写次数；Flash Attention 2优化并行性和warps分配
5. vLLM：PagedAttention管理KV Cache显存（虚拟分页机制），Continuous Batching动态调度请求，吞吐量比HuggingFace高数倍

**延伸考点**：GQA/MLA → 模型压缩 → 推理引擎 → 长上下文优化

### Word2Vec/GloVe [频率: ⭐⭐⭐⭐]

**核心要点**：Word2Vec通过上下文预测词向量（CBOW）或词预测上下文（Skip-gram），GloVe基于全局共现矩阵分解，两者均为静态词向量无法处理多义词。

**常见面试问法**：
- "CBOW和Skip-gram的区别？哪个更适合小数据？"
- "Word2Vec和GloVe的核心区别是什么？"

**回答框架**：
1. CBOW：上下文词预测中心词，训练快，适合大数据；Skip-gram：中心词预测上下文，训练慢但低频词效果好，适合小数据
2. 训练加速：负采样（Negative Sampling，将多分类转为二分类）、层次Softmax（哈夫曼树编码输出层）
3. GloVe：构建全局词-词共现矩阵，目标函数 Σ f(X_ij)(w_i·w_j + b_i + b_j - log X_ij)²，结合全局统计和局部上下文窗口
4. 对比：Word2Vec基于局部上下文窗口（在线学习），GloVe基于全局共现统计（离线分解）；实践中效果相近
5. 局限性：静态词向量，一词一向量无法区分多义；ELMo/BERT通过上下文化解决

**延伸考点**：负采样 → 词向量评估 → 上下文化词向量(ELMo/BERT)

### RNN/LSTM/GRU [频率: ⭐⭐⭐⭐]

**核心要点**：RNN处理序列但存在梯度消失，LSTM引入门控机制（遗忘门/输入门/输出门）解决长程依赖，GRU是LSTM的简化版（更新门/重置门）参数更少。

**常见面试问法**：
- "LSTM是怎么解决梯度消失的？各个门的作用是什么？"
- "LSTM和GRU的区别？怎么选？"

**回答框架**：
1. RNN：h_t = tanh(W_h·h_{t-1} + W_x·x_t + b)，梯度连乘导致消失/爆炸
2. LSTM：遗忘门f_t=σ(W_f·[h_{t-1},x_t])决定丢弃什么；输入门i_t=σ(W_i·[h_{t-1},x_t])决定更新什么；候选记忆C̃_t=tanh(W_c·[h_{t-1},x_t])；细胞状态C_t=f_t⊙C_{t-1}+i_t⊙C̃_t；输出门o_t控制输出
3. 梯度传播：C_t的加法连接使梯度可以无衰减地传递，解决梯度消失
4. GRU：更新门z_t控制新旧状态混合比例，重置门r_t控制前一状态对候选的影响，参数少2/3，训练快
5. 选择：数据少选GRU（参数少不易过拟合），数据多选LSTM（表达能力强），现在大多被Transformer替代

**延伸考点**：梯度消失 → 序列模型 → Attention机制 → Transformer

---

### BERT vs GPT [频率: ⭐⭐⭐⭐⭐]

**核心要点**：BERT是双向编码器通过MLM预训练适合理解任务，GPT是单向解码器通过CLM预训练适合生成任务，两者架构和预训练目标根本不同。

**常见面试问法**：
- "BERT和GPT的核心区别是什么？为什么GPT更适合生成？"
- "BERT为什么不能做生成任务？"

**回答框架**：
1. 架构：BERT是Transformer Encoder（双向自注意力，可见全文），GPT是Transformer Decoder（单向/因果自注意力，只看前文）
2. 预训练目标：BERT用MLM（随机mask 15%token预测）+NSP（下一句预测）；GPT用CLM（自回归预测下一个token）
3. 适用任务：BERT→分类/NER/问答抽取等理解任务；GPT→文本生成/对话/翻译等生成任务
4. BERT不能生成：双向注意力导致训练推理不一致，MLM目标与自回归生成不匹配
5. 发展：BERT→RoBERTa(去NSP,更多数据)→ALBERT(参数共享)；GPT→GPT-2/3/4(规模扩展+RLHF)；T5/BART统一编码解码

**延伸考点**：预训练任务 → 微调策略 → Prompt Tuning → 大模型对齐

---

### 预训练任务 [频率: ⭐⭐⭐⭐]

**核心要点**：MLM是BERT的遮蔽语言模型，CLM是GPT的自回归语言模型，NSP是BERT的句子关系预测，不同预训练任务决定模型能力方向。

**常见面试问法**：
- "MLM和CLM的区别？各适合什么下游任务？"
- "BERT的MLM中15%的mask策略具体是怎么做的？"

**回答框架**：
1. MLM（Masked Language Model）：随机遮蔽15%的token，其中80%替换为[MASK]，10%替换为随机词，10%保持不变；保持输入输出对齐，10%随机和10%不变缓解预训练-微调差异
2. CLM（Causal Language Model）：自回归预测下一个token，P(x_t|x_{<t})，天然适合生成
3. NSP（Next Sentence Prediction）：判断句子B是否是句子A的下一句，RoBERTa发现去掉NSP效果更好（NSP太简单）
4. 其他预训练任务：T5的Span Corruption（遮蔽连续span）、ELECTRA（替换检测，所有token都参与训练效率高）、BART的去噪自编码
5. 预训练数据：数据量、数据质量、去重策略直接影响模型效果

**延伸考点**：BERT → GPT → T5 → 预训练数据工程

---

### Prompt Engineering [频率: ⭐⭐⭐⭐⭐]

**核心要点**：Prompt Engineering通过设计输入提示引导LLM输出期望结果，In-context Learning无需参数更新即可学习新任务，是LLM应用的核心技能。

**常见面试问法**：
- "什么是In-context Learning？它和微调有什么区别？"
- "常见的Prompt技巧有哪些？CoT的原理是什么？"

**回答框架**：
1. Zero-shot：不给示例直接描述任务，简单任务可用
2. Few-shot / In-context Learning：在提示中给几个示例，模型从中学习模式；不更新参数，上下文窗口有限
3. Chain-of-Thought（CoT）：让模型逐步推理，"Let's think step by step"，显著提升数学/逻辑推理能力；Auto-CoT自动生成推理链
4. 其他技巧：角色扮演（"你是一个XX专家"）、输出格式约束（JSON/Markdown）、自我一致性（多次采样取多数）、ReAct（推理+行动循环）
5. 与微调对比：Prompt Engineering零成本快速迭代但受上下文长度限制，微调成本高但可注入领域知识且推理快

**延伸考点**：CoT → Agent → RAG → 微调 vs Prompt

## 四、推荐系统

---

### 协同过滤 [频率: ⭐⭐⭐⭐]

**核心要点**：协同过滤基于"相似用户喜欢相似物品"的假设，User-CF找相似用户推荐其喜欢的物品，Item-CF找相似物品推荐给喜欢相关物品的用户。

**常见面试问法**：
- "User-CF和Item-CF的区别？分别适合什么场景？"
- "协同过滤的冷启动问题怎么解决？"

**回答框架**：
1. User-CF：计算用户相似度（余弦/Jaccard），找到与目标用户最相似的K个用户，推荐这些用户喜欢但目标用户未交互的物品；适合新闻推荐（兴趣驱动、时效性强）
2. Item-CF：计算物品相似度（共现次数），推荐与用户历史偏好物品最相似的物品；适合电商（物品稳定、用户兴趣持久）
3. 相似度计算：余弦相似度、Pearson相关系数、Jaccard系数
4. 冷启动问题：新用户无历史→热门推荐/注册信息/引导选择；新物品无交互→基于内容推荐
5. 缺点：稀疏性问题（交互矩阵极度稀疏）、冷启动、流行度偏差（热门物品被过度推荐）

**延伸考点**：矩阵分解 → 隐语义模型 → 冷启动 → 流行度偏差

---

### 矩阵分解 [频率: ⭐⭐⭐⭐]

**核心要点**：矩阵分解将稀疏的用户-物品交互矩阵R分解为用户矩阵U和物品矩阵V的乘积，用低维隐向量表示用户和物品，ALS和SGD是两种主要求解方法。

**常见面试问法**：
- "矩阵分解和协同过滤的关系？矩阵分解有什么优势？"
- "ALS和SGD求解矩阵分解的区别？"

**回答框架**：
1. 原理：R ≈ U·Vᵀ，用户u对物品i的预测为 p_u·q_i，维度k远小于用户数和物品数
2. 目标函数：min Σ(r_ui - p_u·q_i)² + λ(||p_u||² + ||q_i||²)，正则化防止过拟合
3. ALS（交替最小二乘）：固定V求U，固定U求V，交替迭代至收敛；可并行，适合大数据
4. SGD：随机梯度下降逐条更新 p_u ← p_u + η(e_ui·q_i - λ·p_u)；简单但不易并行
5. 扩展：加入偏置项(r_ui = μ + b_u + b_i + p_u·q_i)、隐式反馈、时间因素；SVD++融合隐式反馈信息

**延伸考点**：协同过滤 → 隐语义模型 → Wide&Deep → 推荐系统全链路

---

### Wide&Deep/DeepFM/DIN [频率: ⭐⭐⭐⭐⭐]

**核心要点**：Wide&Deep结合记忆（线性）和泛化（深度），DeepFM用FM替代人工特征交叉，DIN用注意力机制建模用户兴趣的动态性。

**常见面试问法**：
- "Wide&Deep的Wide部分和Deep部分分别起什么作用？"
- "DeepFM相比Wide&Deep有什么改进？DIN的核心思想是什么？"

**回答框架**：
1. Wide&Deep：Wide部分（交叉特征，记忆能力，记住高频规则）+ Deep部分（Embedding+MLP，泛化能力，发现新组合）；联合训练
2. DeepFM：用FM替代Wide部分，自动学习二阶特征交叉，无需人工构造交叉特征；FM和Deep共享Embedding，端到端训练
3. DIN：用户兴趣是多样的，用注意力机制对用户历史行为加权——与候选物品相关的历史行为权重高；a = softmax(ReLU(W·[item, hist]))·hist
4. DIEN：在DIN基础上加GRU建模兴趣演化，从兴趣提取→兴趣演化
5. 其他模型：DCN(Cross Network显式高阶交叉)、AutoInt(自注意力特征交叉)、BST(Behavior Sequence Transformer)

**延伸考点**：特征交叉 → 注意力机制 → 用户行为建模 → 多任务学习

---

### 多目标优化 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：推荐系统需同时优化点击率、转化率、停留时长等多个目标，ESMM解决转化率样本选择偏差，PLE用CGC结构避免负迁移。

**常见面试问法**：
- "多目标学习的负迁移和跷跷板现象是什么？怎么解决？"
- "ESMM是怎么解决样本选择偏差的？"

**回答框架**：
1. 多目标挑战：负迁移（任务冲突导致共享参数互相干扰）、跷跷板现象（一个目标提升另一个下降）
2. Shared-Bottom：底层共享参数，简单但任务差异大时负迁移严重
3. MMoE：多个Expert网络+门控，每个任务有独立门控选择Expert，软性参数共享
4. ESMM：建模 pCTR × pCTCVR = pCVR，CVR只在点击样本上训练（样本选择偏差），CTCVR在所有曝光样本上可训练，引入pCTR辅助任务
5. PLE（Progressive Layered Extraction）：CGC结构，任务专属Expert + 共享Expert，渐进式多层提取，解决跷跷板现象

**延伸考点**：多任务学习 → 损失加权 → 梯度冲突 → 推荐系统全链路

---

### 召回→粗排→精排→重排 [频率: ⭐⭐⭐⭐⭐]

**核心要点**：推荐系统四阶段漏斗逐级精筛，召回从海量候选中快速取千级子集，粗排轻量模型筛至百级，精排复杂模型精准打分，重排考虑多样性和业务规则。

**常见面试问法**：
- "推荐系统的召回和精排有什么区别？为什么不能只用精排？"
- "多路召回怎么融合？重排要考虑什么？"

**回答框架**：
1. 召回：从全量物品（百万级）中快速召回千级候选，要求速度快；多路召回（协同过滤/向量召回/热门/标签），各路结果合并去重
2. 向量召回：双塔模型（用户塔+物品塔离线计算Embedding），ANN检索（Faiss/HNSW），在线只需向量检索
3. 粗排：轻量模型（双塔/简单DSSM）对千级候选打分筛至百级，平衡速度和精度
4. 精排：复杂模型（DeepFM/DIN/多目标）对百级候选精准打分，综合多目标分数排序
5. 重排：考虑多样性（MMR/DPP）、业务规则（去重/打散/强插）、实时性、公平性；List-wise重排（PRM/SetRank）考虑物品间关系

**延伸考点**：向量召回 → 双塔模型 → 多目标优化 → 推荐系统工程
