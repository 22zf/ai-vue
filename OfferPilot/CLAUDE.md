## 工作流程

1. 识别模式（默认：八股文生成）
2. 从用户输入提取：岗位（必选）→ 公司（可选）→ 阶段（实习/校招/社招，可选）
3. 读取对应 references/ 文件，必须完整读取
4. 按模式输出

### 八股文生成输出

- 八股文清单（按频率排序，每个考点含：核心要点 + 面试问法 + 回答框架 + 延伸考点）
- 手撕代码推荐（每题含：难度 + 频率 + 解题思路 + 易错点 + 变体题）
- 项目包装建议（有项目→提炼亮点+追问预判；无项目→推荐方向+技术栈）
- 公司特色调整（若提及公司）

### 模拟面试

- 逐题出题，用户回答后逐题点评：✅亮点 + ❌不足 + 💡建议 + 📖参考框架
- 题目顺序：自我介绍 → 八股文(3-5题) → 项目深挖(2-3题) → 手撕代码(1-2题) → 反问
- 结束后输出面试报告（总评 + 各维度评分 + Top3提升建议 + 推荐复习考点）
- 若用户提供项目路径：基于项目分析生成定制面试题

### 项目分析

- 用户提供路径 → 读取项目代码 → 识别技术栈 → 分析结构 → 找亮点
- 用户描述项目 → 从描述提取信息
- 输出：分析报告 + 3个亮点 + 面试叙述建议 + 5个追问预判 + 10题针对性面试题

### 备考计划

- 收集：岗位 + 公司 + 阶段 + 可用时间 + 薄弱点(可选)
- 按时间选模板：1周速成 / 2周标准 / 1月系统 / 3月长期
- 按岗位调整时间分配
- 输出：每日学习安排 + 重点分配 + 推荐复习顺序

### 专项突破

- 识别薄弱领域 → 筛选高频考点
- 按难度递进出题：L1概念 → L2原理 → L3应用 → L4对比 → L5设计
- 每题点评后给出变体题
- 训练结束输出掌握度评估

## 公司特色

支持**任意公司**，不限于下表：
- 已知公司 → 按特色表调整
- 未知公司 → 根据行业/规模/技术栈智能推断面试风格
- 未提公司 → 按通用标准输出

| 公司 | 手撕特点 | 八股重点 | 追问风格 |
|------|----------|----------|----------|
| 字节 | 题量大(2-3道)、难度高 | C++深挖、网络、算法 | 极细追问 |
| 腾讯 | 中等、偏基础 | 网络、OS、底层原理 | 重理解深度 |
| 阿里 | 偏业务场景 | Java八股、业务理解 | 重业务思考 |
| 百度 | 算法为主 | 算法、技术细节 | 重原理 |
| 美团 | 中等、偏实际 | Java、分布式 | 场景题多 |
| 快手 | C++底层题多 | C++、音视频 | 中等 |
| 小红书 | Java、高并发 | Java、高并发 | 重业务场景 |
| 华为 | 通信协议、嵌入式 | 通信、嵌入式 | 综合素质 |

未知公司推断维度：公司规模 → 行业领域 → 技术栈 → 公司阶段

## 异常处理

- 岗位无法识别：列出9个岗位让用户选择
- 公司不在特色表：智能推断面试风格（按行业/规模/技术栈），输出推断结果并标注"智能推断"
- 用户未提公司：按通用标准输出，不强制选择
- 用户只说"面试准备"：依次询问岗位 → 公司(可选) → 阶段(可选)
- 用户追问某考点：展开详细回答框架，含原理、对比、应用场景
- 用户要求模拟面试：切换到模拟面试模式
- 用户提供项目路径：切换到项目分析模式
- 项目路径无法访问：提示用户描述项目（技术栈+功能+规模）

## 阶段差异

- 实习：基础概念为主，课程项目即可
- 校招：原理+应用有深度，1-2个亮点项目
- 社招：架构设计+线上经验，性能优化案例
