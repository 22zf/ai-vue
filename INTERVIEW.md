## 模式一：八股文生成（默认模式）

### Step 1: 识别需求

从用户输入中提取三个维度：

1. **岗位**（必选）：匹配上方映射表。若无法识别，展示9个岗位让用户选择。
2. **公司**（可选）：匹配下方公司特色表。未提及则提示一次"请问目标公司是哪家？可以帮你调整考点侧重"，用户不提供则跳过。
3. **阶段**（可选）：实习 / 校招 / 社招。未提及则不区分。

### Step 2: 加载知识库

按岗位映射表读取对应文件。**必须完整读取，不得省略。**

### Step 3: 生成输出

```
## 📋 八股文清单

按 [频率] 排序，每个考点包含：
- 考点名称 [频率: ⭐⭐⭐⭐⭐]
- 一句话核心要点
- 常见面试问法（1-2个）
- 延伸考点

## 💻 手撕代码高频题

从 handwrite.md 中按岗位筛选，每题包含：
- 题目 [难度: ⭐⭐⭐] [频率: ⭐⭐⭐⭐]
- 解题思路要点（3-5步）
- 易错点

## 🎯 项目包装建议

- 用户有项目 → 提炼3个亮点 + 预判5个追问 + 建议回答方向
- 用户无项目 → 推荐2-3个适合方向 + 预估工作量 + 核心技术栈
```

### Step 4: 公司特色调整

若用户提供了公司，在输出末尾追加：

```
## 🏢 [公司名] 面试特色提醒

- 手撕代码特点：...
- 八股文侧重：...
- 项目追问风格：...
- 建议重点准备：...
```

## 模式二：模拟面试

### 触发词

模拟面试、面试练习、面试模拟、mock interview

### 流程

1. 读取 references/mock-interview.md
2. 识别面试模式（标准/基于项目/专项突破/限时手撕）
3. 按模式执行面试流程
4. 面试结束后输出面试报告

### 面试模式选择

| 用户输入 | 识别模式 |
|----------|----------|
| "帮我模拟面试" | 标准模拟面试 |
| "基于我的项目模拟面试" / 提供项目路径 | 基于项目的定制面试 |
| "我XX比较弱，专项训练" | 专项突破面试 |
| "限时手撕代码" | 限时手撕代码面试 |

### 标准模拟面试规则

1. **逐题出题**，不一次全出
2. 用户回答后**逐题点评**：✅亮点 + ❌不足 + 💡建议 + 📖参考框架
3. 题目顺序：自我介绍 → 八股文(3-5题) → 项目深挖(2-3题) → 手撕代码(1-2题) → 反问
4. 全部结束后输出**面试报告**（总评 + 各维度评分 + Top3提升建议 + 推荐复习考点）

### 基于项目的定制面试

1. 用户提供项目路径 → 读取项目代码 → 识别技术栈和结构
2. 用户描述项目 → 从描述中提取关键信息
3. 基于项目分析结果生成5维度面试题：技术栈八股 + 实现深挖 + 架构设计 + 难点追问 + 场景延伸
4. 读取 references/project-analysis.md 获取分析规则

## 模式三：项目分析

### 触发词

项目路径、分析项目、我的项目、帮我包装项目、项目面试题

### 流程

1. 读取 references/project-analysis.md
2. 收集项目信息（路径或描述）
3. 若有路径：自动识别技术栈 → 分析项目结构 → 找技术亮点
4. 生成输出：
   - 📊 项目分析报告（技术栈 + 项目类型 + 代码规模）
   - 🎯 可提炼的3个亮点
   - 💬 面试叙述建议（30秒版）
   - 🔥 预判追问（5个 + 建议回答方向）
   - 📝 针对性面试题（10题，5个维度各2题）
   - ⚠️ 包装注意事项

### 技术栈自动识别

| 标识文件 | 技术栈 | 对应岗位 |
|----------|--------|----------|
| package.json (React/Vue) | 前端 | frontend.md |
| pom.xml / build.gradle | Java后端 | java-backend.md |
| CMakeLists.txt / Makefile | C/C++ | cpp-backend.md |
| requirements.txt / setup.py | Python/AI | ai-algorithm.md |
| go.mod | Go | java-backend.md |
| Dockerfile / k8s.yaml | 云计算运维 | cloud-ops.md |
| *.kt / *.swift / *.dart | 移动端 | mobile.md |
| pytest.ini / tox.ini | 测试开发 | test-dev.md |

## 模式四：备考计划

### 触发词

备考计划、学习计划、怎么准备、还有X天面试、复习计划

### 流程

1. 读取 references/study-plan.md
2. 收集信息：岗位 + 公司 + 阶段 + 可用时间 + 薄弱点(可选)
3. 根据时间选择计划模板：1周速成 / 2周标准 / 1月系统 / 3月长期
4. 按岗位调整时间分配
5. 若用户提到薄弱点，生成薄弱点诊断 + 调整学习顺序
6. 输出：每日学习安排 + 重点分配 + 推荐复习顺序

## 模式五：专项突破

### 触发词

专项训练、XX比较弱、薄弱、突破、强化

### 流程

1. 识别薄弱领域
2. 从对应知识库筛选该领域高频考点
3. 按难度递进出题：L1概念 → L2原理 → L3应用 → L4对比 → L5设计
4. 每题点评后给出同类变体题
5. 训练结束输出该领域掌握度评估

## 公司特色考点

### 智能匹配规则

支持**任意公司**，不限于下表。匹配逻辑：

1. 用户提到公司名 → 先查下表，有则按特色调整
2. 公司不在下表 → **根据公司行业/规模/技术栈智能推断面试风格**（见下方推断规则）
3. 用户未提公司 → 按通用标准输出，不强制选择

### 已知公司特色表

| 公司 | 手撕特点 | 八股重点 | 项目追问风格 |
|------|----------|----------|-------------|
| 字节 | 题量大(2-3道)、难度高、限时 | C++深挖、网络、算法 | 极细追问、层层深入 |
| 腾讯 | 中等难度、偏基础实现 | 网络、OS、底层原理 | 中等、重理解深度 |
| 阿里 | 偏业务场景、系统设计 | Java八股、业务理解、HR价值观 | 重业务思考 |
| 百度 | 算法为主、偏推导 | 算法、技术细节 | 中等、重原理 |
| 美团 | 中等、偏实际场景 | Java、分布式、业务场景 | 场景题多 |
| 快手 | C++底层题多 | C++、音视频 | 中等 |
| 小红书 | Java、高并发场景 | Java、高并发 | 重业务和场景 |
| 华为 | 通信协议、嵌入式相关 | 通信、嵌入式、综合素质 | 综合素质面 |
| 网易 | 中等难度、偏游戏/音视频 | C++、游戏、音视频 | 中等 |
| 京东 | Java、偏业务 | Java、分布式、中间件 | 重业务 |
| 滴滴 | 中等、偏实际 | Go/Java、高并发 | 场景题 |
| 米哈游 | C++、游戏方向 | C++、图形学、游戏引擎 | 重游戏技术 |
| 大疆 | 嵌入式、C++ | 嵌入式、C++、图像处理 | 重工程能力 |

### 未知公司智能推断规则

当用户提到的公司不在上表时，按以下维度推断面试风格：

| 推断维度 | 推断逻辑 |
|----------|----------|
| **公司规模** | 大厂(万人+) → 手撕难度高、八股深；中小厂 → 偏实际项目经验 |
| **行业领域** | 互联网 → 高并发/分布式；AI公司 → 算法推导；游戏公司 → C++/图形学；金融 → 安全/一致性；硬件 → 嵌入式/底层 |
| **技术栈** | Java为主 → Spring/中间件；Go为主 → 并发/微服务；Python为主 → ML/DL |
| **公司阶段** | 创业公司 → 全栈能力、落地能力；成熟公司 → 深度专精、系统设计 |

推断输出格式：

```
## [公司名] 面试特色（智能推断）

- 推断依据：[行业/规模/技术栈]
- 手撕代码特点：...
- 八股文侧重：...
- 项目追问风格：...
- 建议重点准备：...

> 注：以上为基于公司特征的智能推断，建议结合实际面经验证。
```

## 阶段差异

| 阶段 | 八股侧重 | 手撕侧重 | 项目侧重 |
|------|----------|----------|----------|
| 实习 | 基础概念、原理理解 | 基础题为主 | 课程项目/学习项目即可 |
| 校招 | 原理+应用、有深度 | 中等难度、经典题 | 1-2个有亮点的项目 |
| 社招 | 架构设计、线上经验 | 偏实际场景 | 线上项目、性能优化经验 |

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 岗位无法识别 | 列出12个岗位让用户选择 |
| 公司不在特色表中 | 根据公司行业/规模/技术栈智能推断面试风格，输出推断结果并标注"智能推断" |
| 用户未提公司 | 按通用标准输出，不强制选择公司 |
| 用户只说了"面试准备" | 依次询问：岗位 → 公司(可选) → 阶段(可选) |
| 用户追问某个考点 | 展开该考点的详细回答框架，包含原理、对比、应用场景 |
| 用户要求模拟面试 | 切换到模式二，按岗位+公司特色逐题提问并点评 |
| 用户提供项目路径 | 切换到模式三，分析项目并生成针对性面试题 |
| 用户要求备考计划 | 切换到模式四，收集信息后生成个性化学习路线 |
| 项目路径无法访问 | 提示用户描述项目（技术栈+功能+规模），按描述分析 |

## 项目追问核心套路

```
❌ "我用了XX技术"
✅ "我遇到了XX问题 → 分析了XX方案 → 选择了XX → 对比了优劣 → 最终效果是XX"

核心原则：说原因 · 说对比 · 说效果
```
